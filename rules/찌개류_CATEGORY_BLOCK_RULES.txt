## 공통 처리 원칙

- BLOCK_PRIORITY를 위에서부터 적용한다.
- R block 번호 순서가 아니라, 위 BLOCK_PRIORITY에 적힌 순서를 실제 우선순위로 사용한다.
- 이 category는 찌개류 category이며, 전골류도 찌개류 category 내부의 세부 block으로 함께 처리한다.
- 특정 R block에 들어간 뒤에는 해당 block의 INTERNAL_PRIORITY를 위에서부터 적용한다.
- raw_generic_name은 2차 프롬프트에서 생성된 후보명이고, final_generic_name은 3차에서 확정하는 분석용 이름이다.
- final_generic_name 후보 목록은 “강제 통합 가능한 대표명 목록”이다.
- raw_generic_name이 block 안의 final_generic_name 후보로 명확히 통합 가능하면 반드시 해당 final_generic_name으로 통합한다.
- final_generic_name 후보에 없더라도 raw_generic_name이 특정 R block 또는 family_group에 명확히 속하면 새로운 넓은 대표명으로 억지 통합하지 않는다.
- 단, block rule에서 육류찌개, 어패류찌개, 채소찌개, 부속,내장전골류, 육류전골, 어패류전골, 채소전골처럼 명시적으로 fallback final_generic_name을 지정한 경우에는 해당 fallback final_generic_name으로 통합한다.
- R97_찌개류복합검토와 R99_REVIEW는 검토용 block이므로 final_generic_name = raw_generic_name으로 유지한다.
- AI가 새로운 assigned_block이나 family_group을 임의 생성하면 안 된다.
- AI가 CATEGORY_BLOCK_RULES에 없는 넓은 final_generic_name을 임의 생성하면 안 된다.
- 찌개류 category에 속하지 않는 후보는 R98_OUT_OF_SCOPE로 보낸다.
- 단, 찌개 또는 전골 본체가 명확한 후보는 R98_OUT_OF_SCOPE로 보내지 않는다.
- 찌개류 안에는 남을 수 있지만 자동 통합하면 의미 손실이나 category 혼동이 생기는 후보는 R97_찌개류복합검토로 보낸다.
- 찌개류 여부, 음식명 여부, 적용 가능한 block 자체가 불명확하면 R99_REVIEW로 보낸다.
- cuisine_signal은 category rule에서 적극적으로 추론하지 않는다. 대부분 final_generic_name과 assigned_block으로 표현하고, cuisine_signal은 none으로 둬도 된다.


## 주요 우선순위

- 해외찌개류는 한국식 찌개나 전골로 흡수하면 의미가 사라지는 고유 해외식 찌개, 전골, 핫팟 메뉴를 먼저 보호한다.
- 짜글이류는 김치찌개, 된장찌개, 돼지찌개보다 먼저 처리한다.
- 순두부찌개, 비지찌개, 두부찌개는 김치찌개, 된장찌개, 어패류찌개보다 먼저 처리한다.
- 김치찌개, 된장찌개, 청국장, 부대찌개, 고추장찌개는 재료 수식어가 붙어도 기본찌개류로 처리한다.
- 부대전골은 부대찌개가 아니라 R09_육류전골류의 부대전골로 처리한다.
- 두부찌개는 R03_콩,두부찌개류에서 처리한다.
- 두부전골은 두부찌개로 통합하지 않고 R10_채소전골류에서 처리하며 final_generic_name = 두부전골로 보존한다.
- 곱창, 대창, 내장, 막창, 특양, 양대창, 부속 전골 신호가 명확하면 R09_육류전골류보다 R08_부속,내장전골류를 우선한다.
- R08_부속,내장전골류는 곱창전골, 대창전골, 내장전골을 먼저 살리고, 그 외 명확한 부속,내장 기반 전골은 final_generic_name = 부속,내장전골류로 통합한다.
- 단, “양” 한 글자만으로 부속,내장전골류로 판단하지 않는다.
- 만두전골은 김치전골, 버섯전골, 두부전골보다 먼저 R11_만두전골류에서 처리한다.
- 김치만두전골, 두부만두전골, 버섯만두전골은 채소전골류가 아니라 만두전골류를 우선한다.
- 육류전골류는 수육전골, 불고기전골, 부대전골을 먼저 살리고, 그 외 일반 육류 기반 전골은 final_generic_name = 육류전골로 통합한다.
- 어패류전골류는 조개전골, 낙지전골, 동태전골, 해물전골을 먼저 살리고, 그 외 어패류 기반 전골은 final_generic_name = 어패류전골로 통합한다.
- 채소전골류는 김치전골, 버섯전골, 두부전골을 먼저 살리고, 그 외 채소 기반 전골은 final_generic_name = 채소전골로 통합한다.
- 훠궈 샤브샤브처럼 훠궈와 샤브샤브가 함께 있으면 훠궈를 우선한다.
- 마라탕 단독은 R98_OUT_OF_SCOPE로 보내지만, 마라전골 또는 마라탕전골은 R01_해외찌개류에서 처리한다.
- 해물전골은 raw_generic_name이 해물전골 또는 해산물전골 계열일 때만 final_generic_name = 해물전골로 처리한다.
- 조개, 낙지, 동태 등 명확한 어패류 전골 anchor가 있으면 해물전골보다 해당 anchor를 우선한다.
- 곱도리탕은 절대 R97_찌개류복합검토로 보내지 않고 R98_OUT_OF_SCOPE로 보낸다.


## R98_OUT_OF_SCOPE

family_group = ""

목적:
찌개류 category에 속하지 않거나 다른 category가 더 적절한 후보를 분리한다.

매칭 조건:
- 매운탕, 감자탕, 알탕, 마라탕, 닭매운탕, 참게메기매운탕처럼 탕류 본체
- 닭볶음탕, 닭도리탕, 곱도리탕처럼 볶음탕 또는 탕 본체가 강한 후보
- 김치찜, 돼지김치찜, 갈비김치찜, 갈비찜뚝배기처럼 찜류 본체
- 볶음, 구이, 튀김, 전, 부침, 무침, 회, 숙회처럼 찌개류 category가 아닌 조리법 본체
- 국밥, 덮밥, 비빔밥, 면, 칼국수, 라면, 파스타, 샐러드처럼 다른 메뉴 본체가 명확한 후보
- 정식, 세트, 코스, 한정식처럼 구성 상품 본체가 강하고 찌개 또는 전골 단품 본체가 불명확한 후보

- 찌개 또는 전골 본체가 불명확한 정식, 세트, 코스는 R98_OUT_OF_SCOPE 또는 R97_찌개류복합검토로 보낸다.

## R01_해외찌개류

family_group = 해외찌개류

final_generic_name 후보:
- 훠궈
- 스키야끼
- 나베
- 샤브샤브
- 마라전골

INTERNAL_PRIORITY:
1. 훠궈
2. 스키야끼
3. 나베
4. 샤브샤브
5. 마라전골

bucket 판단:
- 해외찌개류는 한국식 찌개나 전골로 흡수하면 의미가 사라지는 고유 해외식 찌개, 전골, 핫팟 메뉴를 처리한다.
- 훠궈는 마라전골, 샤브샤브보다 우선한다.
- 스키야끼, 스키야키, 스끼야끼는 final_generic_name = 스키야끼로 통합한다.
- 나베는 일본식 전골 또는 냄비요리 신호가 강하므로 한국식 전골로 흡수하지 않는다.
- 샤브샤브는 해외찌개류로 처리하되, 샤브만두전골처럼 만두전골 본체가 명확하면 R11_만두전골류를 우선한다.
- 마라전골은 final_generic_name = 마라전골로 통합한다.
- 마라탕전골은 마라탕 단독이 아니라 전골 표현이 명확하므로 final_generic_name = 마라전골로 통합한다.
- 단, 훠궈 신호가 있으면 마라전골보다 훠궈를 우선한다.
- 예시에 없는 해외 찌개 또는 전골 신호가 명확하되 후보 목록 중 하나로 확정하기 어려우면 final_generic_name = raw_generic_name으로 유지하고 needs_review로 둔다.

통합 규칙:
- 훠궈, 훠궈전골, 마라훠궈, 마라훠궈전골, 미니훠궈, 반반탕훠궈, 꼬챙이훠궈, 훠궈 샤브샤브
  → final_generic_name = 훠궈

- 스키야끼, 스키야키, 스끼야끼, 관서식 스키야키, 관서풍 스키야키, 소고기 스키야끼
  → final_generic_name = 스키야끼

- 나베, 오뎅나베, 오댕나베, 간사이 오뎅나베, 가츠나베, 곱창나베, 규스지나베, 김치나베, 밀푀유나베, 모츠나베
  → final_generic_name = 나베

- 샤브샤브, 샤브, 소고기샤브샤브, 곱창샤브샤브, 샤브전골, 김치샤브전골, 대창샤브전골
  → final_generic_name = 샤브샤브

- 마라전골, 대창마라전골, 우삼겹마라전골, 꼬치마라전골, 마라탕전골
  → final_generic_name = 마라전골

주의 규칙:
- 훠궈는 샤브샤브보다 우선한다.
- 마라훠궈전골은 마라전골이 아니라 훠궈로 처리한다.
- 마라탕 단독은 R98_OUT_OF_SCOPE로 보낸다.
- 마라탕전골은 R01_해외찌개류의 마라전골로 처리한다.
- 샤브만두전골, 샤브샤브만두전골은 만두전골 본체가 명확하면 R11_만두전골류로 보낸다.
- 나베는 일반 전골류로 보내지 않는다.
- 해외 고유 전골명인지 한국식 전골명인지 불명확하면 R97_찌개류복합검토로 보낸다.

attribute 규칙:
- 마라, 반반탕, 소고기, 곱창, 대창, 김치, 간사이, 밀푀유, 모츠, 오뎅, 샤브 등 세부 재료와 스타일은 attribute에 남긴다.


## R02_짜글이류

family_group = 짜글이류

final_generic_name 후보:
- 짜글이

INTERNAL_PRIORITY:
1. 짜글이

bucket 판단:
- 짜글이류는 김치찌개, 돼지찌개, 된장찌개보다 먼저 처리한다.
- 김치짜글이, 돼지짜글이, 갈비짜글이처럼 재료가 달라도 final_generic_name = 짜글이로 통합한다.
- 짜글이, 자글이 표기 변형은 짜글이로 통합한다.
- 예시에 없는 짜글이 후보도 짜글이 신호가 명확하면 final_generic_name = 짜글이로 처리한다.

통합 규칙:
- 짜글이, 자글이, 김치짜글이, 돼지짜글이, 돼지고기짜글이, 갈비짜글이, 곱짜글이, 김치순두부짜글이, 갈치짜글이
  → final_generic_name = 짜글이

주의 규칙:
- 짜글이는 김치찌개, 돼지찌개, 어패류찌개보다 우선한다.
- 짜글이인지 단순 찌개인지 불명확하면 R97_찌개류복합검토로 보낸다.

attribute 규칙:
- 김치, 돼지, 갈비, 곱창, 순두부, 갈치 등 세부 재료는 attribute에 남긴다.


## R03_콩,두부찌개류

family_group = 콩,두부찌개류

final_generic_name 후보:
- 순두부찌개

- 비지찌개

INTERNAL_PRIORITY:
1. 순두부찌개

5. final_generic_name 후보로 자동 통합하면 의미 손실이 큰 후보

매칭 조건:
- 낙곱새, 낙곱전골처럼 낙지, 곱창, 새우가 결합되어 부속,내장전골류 또는 어패류전골류 중 하나로 자동 통합하기 어려운 후보
- 등갈비해물전골, 닭낙지전골, 불고기낙지전골처럼 육류전골과 어패류전골이 강하게 결합된 후보
- 훠궈샤브전골, 마라샤브전골처럼 해외전골과 국내식 전골 표현이 결합된 후보
- 전골 단독, 찌개 단독처럼 재료 축이 없는 후보
- 예시에 없고 찌개 또는 전골 신호는 있으나 R01~R12 중 어느 축으로 보낼지 확정하기 어려운 후보
- 찌개류 안에 남을 가능성은 있지만 자동 통합하면 의미 손실이 큰 후보
- 재료명은 있으나 찌개 또는 전골 본체가 불명확한 후보

제외 조건:
- 해외찌개류 신호가 명확하면 R01
- 짜글이 신호가 명확하면 R02
- 콩,두부찌개류 신호가 명확하면 R03
- 기본찌개류 신호가 명확하면 R04
- 채소찌개류 신호가 명확하면 R05
- 육류찌개류 신호가 명확하면 R06
- 어패류찌개류 신호가 명확하면 R07
- 부속,내장전골류 신호가 명확하면 R08
- 만두전골류 신호가 명확하면 R11
- 육류전골류 신호가 명확하면 R09
- 어패류전골류 신호가 명확하면 R12
- 채소전골류 신호가 명확하면 R10
- 두부전골은 R10_채소전골류의 두부전골로 보낸다.
- 막창전골, 특양전골, 양대창전골, 부속전골은 R08_부속,내장전골류의 부속,내장전골류 fallback으로 보낸다.
- 곱창, 대창, 내장, 막창, 특양, 양대창, 부속 신호가 명확한 전골은 R09_육류전골류가 아니라 R08_부속,내장전골류로 보낸다.
- 육류 기반 전골은 R09_육류전골류의 육류전골 fallback으로 보낸다.
- 어패류 기반 전골은 R12_어패류전골류의 어패류전골 fallback으로 보낸다.
- 채소 기반 전골은 R10_채소전골류의 채소전골 fallback으로 보낸다.
- 육류 기반 찌개는 R06_육류찌개류의 육류찌개 fallback으로 보낸다.
- 어패류 기반 찌개는 R07_어패류찌개류의 어패류찌개 fallback으로 보낸다.
- 채소 기반 찌개는 R05_채소찌개류의 채소찌개 fallback으로 보낸다.
- 찌개류 category가 아님이 명확하면 R98
- 곱도리탕은 R97이 아니라 R98_OUT_OF_SCOPE로 보낸다.
- 찌개류 여부 자체가 불명확하면 R99

출력 규칙:
- assigned_block = R97_찌개류복합검토
- final_generic_name = raw_generic_name
- family_group = 찌개류복합검토
- normalization_actions = ["broad_merge_review"]
- review_status = needs_review


## R99_REVIEW

family_group = ""

목적:
찌개류 여부, 음식명 여부, 또는 적용 가능한 block 자체가 불명확한 후보를 최종 검토한다.

매칭 조건:
- 어떤 block에도 안정적으로 들어가지 않는 경우
- raw_generic_name만으로 찌개류인지 알 수 없는 경우
- 음식명인지 옵션, 소스, 재료명인지 불명확한 경우
- 오타가 심해 찌개류 여부 판단이 어려운 후보
- 브랜드명 또는 업장 고유명처럼 보이나 찌개류 본체가 드러나지 않는 후보
- 재료가 불명확하거나 어느 찌개류 축에도 확실히 속하지 않는 후보

출력 규칙:
- assigned_block = R99_REVIEW
- final_generic_name = raw_generic_name
- family_group = ""
- normalization_actions = ["broad_merge_review"]
- review_status = needs_review
