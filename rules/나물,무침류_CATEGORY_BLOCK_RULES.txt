# 1. 공통 처리 원칙

- BLOCK_PRIORITY를 위에서부터 적용한다.
- 특정 R block에 들어간 뒤에는 해당 block의 INTERNAL_PRIORITY를 위에서부터 적용한다.
- raw_generic_name은 2차 프롬프트에서 생성된 후보명이고, final_generic_name은 3차에서 확정하는 분석용 이름이다.
- final_generic_name 후보 목록은 “강제 통합 가능한 대표명 목록”이다.
- raw_generic_name이 block 안의 final_generic_name 후보로 명확히 통합 가능하면 반드시 해당 final_generic_name으로 통합한다.
- final_generic_name 후보에 없더라도, raw_generic_name이 특정 R block/family_group에 명확히 속하면 새로운 넓은 final_generic_name을 임의 생성하지 않는다.
- 이 경우 final_generic_name = raw_generic_name으로 유지하고, family_group은 해당 R block의 family_group으로 출력하며, normalization_actions = ["broad_merge_review"], review_status = needs_review로 둔다.
- R97_나물무침류복합검토와 R99_REVIEW는 검토용 block이므로 final_generic_name = raw_generic_name으로 유지한다.
- AI가 새로운 assigned_block, family_group, final_generic_name을 임의 생성하면 안 된다.
- 예시는 실제 데이터에 등장한 clean_text 기반으로만 작성한다.
- 데이터에 없는 가상 메뉴명은 예시로 쓰지 않는다.


# 2. 분석 목적 기준

이 프롬프트의 목적은 메뉴 조합 분석용 final_generic_name을 만드는 것이다.

따라서 세부 재료명을 모두 final_generic_name으로 살리는 것이 목적이 아니다.

- 반복 빈도가 낮고 재료명만 다른 무침류는 더 넓은 final_generic_name으로 흡수한다.
- 단, 업태/조합 신호가 강한 고유 메뉴명은 별도 final_generic_name으로 보존한다.
- 세부 재료명은 final_generic_name으로 쪼개지 말고 attribute에 남긴다.
- 소면, 쫄면, 야채, 미나리, 파채, 오이, 초무침, 매운, 마라, 들깨, 간장 같은 부가 수식어는 final_generic_name을 새로 만들지 않는다.
- 주재료 block이 명확하면 부가 수식어는 attribute로 보존한다.


# 3. 공통 우선 처리 규칙

- 나물,무침류가 아닌 후보는 R98_OUT_OF_SCOPE로 보낸다.
- 샐러드 신호가 명확하면 R01_샐러드를 우선한다.
- 냉채 또는 양장피 신호가 명확하면 R02_냉채류를 우선한다.
- 해외 고유 무침/쌈 신호가 명확하면 R03_해외무침으로 보낸다.
- 어패류 재료와 무침 신호가 함께 있으면 R05_어패류무침을 우선한다.
- 육류 재료와 무침 신호가 함께 있으면 R06_육류무침을 우선한다.
- 채소무침은 최종 흡수 block으로 가장 뒤에서 처리한다.
- 대부분 쫄면,소면이 같이 무침으로 곁들어진 것은 사이드인 확률이 높기 때문에 쫄면,소면은 배제하고 남는 음식 신호들만 축을 판단한다. 

# 4. 주요 통합 방향

## 냉채류

- 양장피는 final_generic_name = 양장피로 보존한다.
- 그 외 냉채류는 final_generic_name = 냉채로 흡수한다.
- 냉채 내부의 재료명은 attribute에 남긴다.

## 해외무침

- 월남쌈, 오이탕탕이, 건두부무침만 final_generic_name으로 보존한다.
- 건두부무침은 실제 조리 성격상 냉채성일 수 있으나, 메뉴명에 `냉채`가 명시되지 않은 경우 R03_해외무침에서 건두부무침으로 보존한다.
- `건두부냉채`, `냉채 오이건두부무침`처럼 냉채 신호가 있으면 R02_냉채류가 우선한다.

## 어패류무침

- 회무침만 final_generic_name = 회무침으로 보존한다.
- 회무침을 제외한 골뱅이, 오징어, 꼬막, 굴, 낙지, 문어, 소라, 우렁, 갑오징어 계열은 final_generic_name = 어패류무침으로 흡수한다.
- 골뱅이소면무침, 골뱅이쫄면무침처럼 소면/쫄면이 붙어도 final_generic_name은 어패류무침으로 흡수한다.
- 소면/쫄면 여부는 attribute에 남긴다.

## 육류무침
- 수육무침과 양깃머리무침만 별도 final_generic_name으로 보존한다.
- 그 외 차돌, 소고기, 돼지귀, 닭, 닭껍질, 똥집, 우삼겹, 흑염소, 깐양, 내장 계열은 final_generic_name = 육류무침으로 흡수한다.

## 채소무침
- 묵무침만 별도 final_generic_name으로 보존한다.
- 두부김치는 무침류가 아니므로 R04_채소무침에서 보존하지 않는다.
- 두부김치는 R98_OUT_OF_SCOPE로 보낸다.
- 그 외 오이, 도라지, 미역, 목이버섯, 미나리, 콩나물, 고수, 양파, 감자, 가지, 고사리, 더덕, 다시마 계열은 final_generic_name = 채소무침으로 흡수한다.

- 건두부무침은 실제 조리 성격상 냉채성 중국식 cold dish일 수 있다.
- 그러나 이 taxonomy에서는 메뉴명에 `냉채`가 명시되지 않은 경우 R02_냉채류로 흡수하지 않는다.
- `건두부 + 무침` 후보는 중식/마라/양꼬치 계열 사이드 신호를 보존하기 위해 final_generic_name = 건두부무침으로 둔다.
- 건두부냉채, 냉채 오이건두부무침처럼 냉채 단어가 있으면 R02_냉채류가 우선한다.
- 홍유 건두부처럼 무침/냉채 신호가 없고 조리 형태가 불명확하면 R97_나물무침류복합검토로 보낸다.

## 기타 주의 규칙

- 샐러드 신호가 명확하면 R01_샐러드를 우선한다.
- 냉채 신호가 명확하면 R02_냉채류를 우선한다.
- 모닝글로리, 공심채, 푸루공심채는 일반적으로 볶음 요리로 인식되므로 R98_OUT_OF_SCOPE로 이동한다. 
- 과카몰리 단독/살사형 후보와 타코와사비 계열은 R98_OUT_OF_SCOPE로 보낸다.
- 과카몰리샐러드처럼 샐러드가 본체인 후보는 R01_샐러드로 보낸다.

## attribute 규칙

- 월남쌈, 오이탕탕이, 마라, 건두부, 오이  등 고유 음식명과 재료명은 attribute에 남긴다.

- 푸주무침처럼 해외무침 final_generic_name 후보에서는 제외했지만 추후 별도 축 검토가 필요한 후보

- 홍유 건두부처럼 무침/냉채 신호가 없고 조리 형태가 불명확한 후보

- 채소/어패류/육류 중 주재료 축이 불명확한 무침류
- 나물,무침류 안에 남을 가능성은 있지만 자동 통합하면 의미 손실이 큰 후보

## 제외 조건

- 샐러드 신호가 명확하면 R01_샐러드
- 냉채/양장피 신호가 명확하면 R02_냉채류
- 해외무침 신호가 명확하면 R03_해외무침
- 어패류무침 신호가 명확하면 R05_어패류무침
- 육류무침 신호가 명확하면 R06_육류무침
