# CATEGORY_BLOCK_RULES: 전/부침류

## BLOCK_PRIORITY

R98_OUT_OF_SCOPE  
R01_해외부침류  
R02_계란부침류  
R03_전병류  
R04_대표전류  
R05_육류전  
R06_어패류전  
R07_채소전  
R97_전부침류복합검토  
R99_REVIEW  


## 공통 처리 원칙

- BLOCK_PRIORITY를 위에서부터 적용한다.
- 특정 R block에 들어간 뒤에는 해당 block의 INTERNAL_PRIORITY를 위에서부터 적용한다.
- raw_generic_name은 2차 프롬프트에서 생성된 후보명이고, final_generic_name은 3차에서 확정하는 분석용 이름이다.
- final_generic_name 후보 목록은 “강제 통합 가능한 대표명 목록”이다.
- raw_generic_name이 block 안의 final_generic_name 후보로 명확히 통합 가능하면 반드시 해당 final_generic_name으로 통합한다.
- final_generic_name 후보에 없더라도, raw_generic_name이 특정 R block/family_group에 명확히 속하면 새로운 넓은 대표명으로 억지 통합하지 않는다.
- 다만 R05_육류전, R06_어패류전, R07_채소전은 흡수형 block이므로, block 내부 통합 규칙에 명시된 후보는 각각 육류전/어패류전/채소전으로 통합할 수 있다.
- 이 경우 세부 재료명은 final_generic_name으로 새로 만들지 않고 attribute에 남긴다.
- R97_전부침류복합검토와 R99_REVIEW는 검토용 block이므로 final_generic_name = raw_generic_name으로 유지한다.
- AI가 새로운 assigned_block이나 family_group을 임의 생성하면 안 된다.
- AI가 CATEGORY_BLOCK_RULES에 없는 넓은 final_generic_name을 임의 생성하면 안 된다.
- 전/부침류가 아닌 후보는 R98_OUT_OF_SCOPE로 보낸다.
- 해외부침류, 계란부침류, 전병류, 대표전류는 육류전/어패류전/채소전보다 먼저 처리한다.
- 대표전류는 재료 기반 흡수 block보다 먼저 처리한다.
- 해물파전은 파전으로 통합하지 않고 final_generic_name = `해물파전`으로 보존한다.
- 파전은 해물파전과 분리하여 final_generic_name = `파전`으로 보존한다.
- 녹두전과 빈대떡은 같은 축으로 보고 final_generic_name = `녹두전`으로 통합한다.
- 육전, 동그랑땡, 완자전은 육류전 안에서 각각 별도 final_generic_name으로 보존한다.
- 대표전류에 해당하지 않는 나머지 육류 기반 전은 final_generic_name = `육류전`으로 통합한다.
- 대표전류에 해당하지 않는 나머지 어패류 기반 전은 final_generic_name = `어패류전`으로 통합한다.
- 대표전류에 해당하지 않는 나머지 채소 기반 전은 final_generic_name = `채소전`으로 통합한다.
- 지짐이/지짐/부침개는 전/부침류 안에 남을 수 있지만, 특정 final_generic_name으로 확정되지 않으면 R97_전부침류복합검토로 보낸다.
- 전/부침류 여부, 음식명 여부, 적용 가능한 block 자체가 불명확하면 R99_REVIEW로 보낸다.

## R98_OUT_OF_SCOPE

family_group = ""

목적:  
전/부침류가 아니거나 다른 category가 더 적절한 후보를 분리한다.

매칭 조건:
- 볶음, 구이, 찜, 탕, 국, 찌개, 전골, 조림, 튀김, 무침, 회처럼 전/부침류가 아닌 조리법 본체
- 계란찜, 달걀찜처럼 찜류 본체
- 만두, 튀김만두, 군만두처럼 만두류 본체
- 덮밥, 비빔밥, 김밥, 초밥, 면, 파스타, 샐러드처럼 다른 메뉴 본체가 명확한 후보
- 소스명, 드레싱명, 토핑명, 추가, 옵션, 세트, 코스, 비메뉴 설명문

출력 규칙:
- assigned_block = R98_OUT_OF_SCOPE
- final_generic_name = raw_generic_name
- family_group = ""
- normalization_actions = ["exclude_or_correction"]
- review_status = correction_candidate


## R01_해외부침류
family_group = 해외부침류

final_generic_name 후보:
- 오코노미야끼
- 몬자야끼
- 프리타타

INTERNAL_PRIORITY:
1. 오코노미야끼
2. 몬자야끼
3. 프리타타

bucket 판단:
- 해외부침류는 한국식 전/부침류로 흡수하면 의미가 사라지는 해외식 또는 고유 부침류를 먼저 처리하는 block이다.
- 오코노미야끼와 히로시마야키는 같은 축으로 보고 final_generic_name = `오코노미야끼`로 통합한다.
- 몬자야끼는 오코노미야끼로 합치지 않고 final_generic_name = `몬자야끼`로 보존한다.
- 프리타타는 계란부침류로 흡수하지 않고 final_generic_name = `프리타타`로 보존한다.
- 해외부침류 신호는 명확하지만 아래 final_generic_name 후보로 통합하기 어렵다면 final_generic_name = raw_generic_name으로 유지하고 needs_review로 둔다.

통합 규칙:
- 오코노미야끼, 오꼬노미야끼, 오코노미야키, 오꼬노미야키, 히로시마야끼, 히로시마야키, 히로시마오코노미야키, 히로시마풍오코노미야끼, okonomiyaki  
  → final_generic_name = 오코노미야끼

- 몬자야끼, 몬자야키, monjayaki  
  → final_generic_name = 몬자야끼

- 프리타타, frittata  
  → final_generic_name = 프리타타

주의 규칙:
- 오코노미야끼/몬자야끼는 채소전/해물전으로 보내지 않는다.
- 프리타타는 계란부침류로 보내지 않는다.
- 해외부침류인지 일반 전/부침류인지 불명확하면 R97_전부침류복합검토로 보낸다.

attribute 규칙:
- 히로시마, 해물, 치즈, 고기, 야채, 프리타타 등 세부 재료와 스타일은 attribute에 남긴다.

## R02_계란부침류

family_group = 계란부침류

final_generic_name 후보:
- 오믈렛
- 계란말이
- 계란후라이

INTERNAL_PRIORITY:
1. 오믈렛
2. 계란말이
3. 계란후라이

bucket 판단:
- 계란부침류는 계란을 주재료로 한 부침/전/팬 조리 후보를 처리하는 block이다.
- 오믈렛은 계란말이/계란후라이로 통합하지 않고 final_generic_name = `오믈렛`으로 보존한다.
- 계란말이 계열은 final_generic_name = `계란말이`로 통합한다.
- 계란후라이 계열은 final_generic_name = `계란후라이`로 통합한다.
- 계란 기반이지만 프리타타 신호가 명확하면 R01_해외부침류로 보낸다.
- 계란부침류는 명확하지만 아래 final_generic_name 후보로 확정하기 어렵다면 final_generic_name = raw_generic_name으로 유지하고 needs_review로 둔다.

통합 규칙:
- 오믈렛, 오믈렛트, 오믈레트, omelette, omelet  
  → final_generic_name = 오믈렛
- 계란말이, 달걀말이, 야채계란말이, 치즈계란말이, 명란계란말이, 다시마끼, 다시마키, 다마고야끼, 타마고야끼  
  → final_generic_name = 계란말이
- 계란후라이, 달걀후라이, 계란프라이, 달걀프라이, fried egg  
  → final_generic_name = 계란후라이

주의 규칙:
- 계란찜/달걀찜은 찜류 본체이므로 R98_OUT_OF_SCOPE로 보낸다.
- 프리타타는 R01_해외부침류 우선이다.
- 계란전이라는 표현이 있고 다른 대표전 신호가 없으면 계란부침류에서 needs_review로 둘 수 있다.

attribute 규칙:
- 야채, 치즈, 명란, 다시마끼, 타마고야끼, 프라이 등 세부 재료와 스타일은 attribute에 남긴다.

## R03_전병류

family_group = 전병류

final_generic_name 후보:
- 전병

INTERNAL_PRIORITY:
1. 전병

bucket 판단:
- 전병류는 전병이라는 형태가 본체인 후보를 처리한다.
- 메밀전병, 김치전병, 고기전병처럼 속재료가 달라도 final_generic_name = `전병`으로 통합한다.
- 전병 신호가 명확하면 대표전류/육류전/어패류전/채소전보다 먼저 처리한다.

통합 규칙:
- 전병, 메밀전병, 김치전병, 고기전병, 감자전병, 해물전병  
  → final_generic_name = 전병

주의 규칙:
- 전병 신호가 명확하면 김치전/감자전/육류전/어패류전/채소전으로 보내지 않는다.
- 전병인지 일반 전인지 불명확하면 R97_전부침류복합검토로 보낸다.

attribute 규칙:
- 메밀, 김치, 고기, 감자, 해물 등 속재료와 수식어는 attribute에 남긴다.


## R04_대표전류

family_group = 대표전류

final_generic_name 후보:
- 모듬전
- 녹두전
- 해물파전
- 파전
- 부추전
- 김치전
- 감자전

INTERNAL_PRIORITY:
1. 모듬전
2. 녹두전
3. 해물파전
4. 파전
5. 부추전
6. 김치전
7. 감자전

bucket 판단:
- 대표전류는 육류전/어패류전/채소전으로 흡수하기 전에 먼저 살려야 하는 고유 전 이름을 처리하는 block이다.
- 모듬전은 구성 상품이므로 다른 재료명보다 우선한다.
- 녹두전과 빈대떡은 같은 축으로 보고 final_generic_name = `녹두전`으로 통합한다.
- 해물파전은 파전으로 통합하지 않고 final_generic_name = `해물파전`으로 보존한다.
- 해물/해산물/오징어/굴/새우/낙지/쭈꾸미/조개 등 어패류 재료 + 파전 구조는 final_generic_name = `해물파전`으로 통합한다.
- 어패류 재료가 없는 파전 계열은 final_generic_name = `파전`으로 통합한다.
- 부추전, 김치전, 감자전은 각각 별도 final_generic_name으로 보존한다.
- 단,해물부추전, 오징어김치전, 명란감자전처럼 해물파전과 부추전 같은 복합전이어도 대중적으로 해물과 명란이 부재료로 들어감이 인식되면 부추전/김치전/감자전 신호가 명확하면 해당 대표전류로 통합한다.
- 대표전류에 해당하지 않는 나머지 전은 뒤의 육류전/어패류전/채소전에서 판단한다.
- 대표전류는 명확하지만 아래 final_generic_name 후보로 확정하기 어렵다면 final_generic_name = raw_generic_name으로 유지하고 needs_review로 둔다.

통합 규칙:
- 모듬전, 모둠전, 전모듬, 전모둠, 수제모듬전, 삼색모듬전, 즉석모듬전  
  → final_generic_name = 모듬전

- 녹두전, 녹두빈대떡, 빈대떡, 녹두지짐이, 김치녹두전, 고기녹두전, 녹두해물전  
  → final_generic_name = 녹두전

- 해물파전, 해산물파전, 오징어파전, 굴파전, 새우파전, 쭈꾸미파전, 주꾸미파전, 낙지파전, 조개파전  
  → final_generic_name = 해물파전

- 파전, 대파전, 쪽파전, 실파전, 대파부침개  
  → final_generic_name = 파전

- 부추전, 해물부추전, 새우부추전, 오징어부추전, 굴부추전, 미나리부추전, 가자미부추전  
  → final_generic_name = 부추전

- 김치전, 해물김치전, 오징어김치전, 굴김치전, 낙지김치전, 고기김치전, 치즈김치전  
  → final_generic_name = 김치전

- 감자전, 치즈감자전, 명란감자전, 베이컨감자전, 새우감자전, 부추감자전  
  → final_generic_name = 감자전

주의 규칙:
- 대표전류는 육류전/어패류전/채소전보다 먼저 적용한다.
- 해물파전은 파전으로 통합하지 않는다.
- 해물파전은 어패류전보다 먼저 적용한다.
- 파전은 채소전보다 먼저 적용한다.
- 해물파전, 오징어파전, 굴파전, 새우파전은 어패류전이 아니라 해물과 파의 형태가 포함된 해물파전이다.
- 오징어김치전은 어패류전이 아니라 김치전이다.
- 명란감자전은 어패류전이 아니라 감자전이다.
- 새우부추전은 어패류전이 아니라 부추전이다.
- 김치전은 김치류 correction이 아니라 전/부침류 안의 대표전류로 유지한다.
- 전병 신호가 명확하면 R03_전병류를 우선한다.
- 오코노미야끼/몬자야끼/프리타타 신호가 명확하면 R01_해외부침류로 보낸다.

attribute 규칙:
- 해물, 오징어, 굴, 새우, 쭈꾸미, 낙지, 조개, 김치, 치즈, 명란, 베이컨, 고기, 미나리, 부추 등 세부 재료와 수식어는 attribute에 남긴다.


## R05_육류전

family_group = 육류전

final_generic_name 후보:
- 육전
- 동그랑땡
- 완자전
- 육류전

INTERNAL_PRIORITY:
1. 육전
2. 동그랑땡
3. 완자전
4. 육류전

bucket 판단:
- 육류전은 육류/고기/부속 재료가 본체인 전류를 처리하는 흡수형 block이다.
- 육전은 별도 final_generic_name으로 보존한다.
- 동그랑땡은 별도 final_generic_name으로 보존한다.
- 완자전은 동그랑땡으로 합치지 않고 별도 final_generic_name = `완자전`으로 보존한다.
- 육전/동그랑땡/완자전에 해당하지 않는 나머지 육류 기반 전은 family-level로 final_generic_name = `육류전`으로 흡수 통합한다.
- 이때 세부 육류명은 final_generic_name으로 새로 만들지 않고 attribute에 남긴다.
- 대표전류에 해당하면 R04_대표전류를 우선한다.

통합 규칙:

## R97_전부침류복합검토

family_group = 전부침류복합검토

목적:  
전/부침류 안에는 남을 수 있지만, 기존 R01~R07의 family_group 또는 final_generic_name 후보로 자동 통합하면 의미 손실이나 category 혼동이 생길 수 있는 후보를 검토용으로 분리한다.

INTERNAL_PRIORITY:
1. 전/부침류 안에 남을 수 있지만 조리 형태가 애매한 후보
2. 지짐이/지짐/부침개처럼 특정 대표명으로 확정하기 어려운 후보
3. 대표전류/육류전/어패류전/채소전 중 어느 축인지 불명확한 후보
4. 세트/모듬/구성 상품 성격이 강한 후보 중 자동 통합이 위험한 후보
5. final_generic_name 후보로 자동 통합하면 의미 손실이 큰 후보

매칭 조건:
- 지짐이, 지짐, 부침, 부침개처럼 전/부침류 안에 남을 수 있으나 특정 final_generic_name으로 확정하기 어려운 후보
- 메밀부침, 수수부꾸미, 부꾸미처럼 전병/채소전/대표전류 중 어디로 보낼지 애매한 후보
- 미니전, 전 2장처럼 구성/수량 정보가 본체처럼 보이는 후보
- 전/부침류는 맞지만 대표전류/육류전/어패류전/채소전 중 어디로 보낼지 불명확한 후보
- 전/부침류 안에 남을 가능성은 있지만 자동 통합하면 의미 손실이 큰 후보

제외 조건:
- 해외부침류 신호가 명확하면 R01
- 계란부침류 신호가 명확하면 R02
- 전병 신호가 명확하면 R03
- 대표전류 신호가 명확하면 R04
- 육류전 신호가 명확하면 R05
- 어패류전 신호가 명확하면 R06
- 채소전 신호가 명확하면 R07
- 전/부침류가 아님이 명확하면 R98
- 전/부침류 여부 자체가 불명확하면 R99

출력 규칙:
- assigned_block = R97_전부침류복합검토
- final_generic_name = raw_generic_name
- family_group = 전부침류복합검토
- normalization_actions = ["broad_merge_review"]
- review_status = needs_review
