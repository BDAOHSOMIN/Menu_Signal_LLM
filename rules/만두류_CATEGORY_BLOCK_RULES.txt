# CATEGORY_BLOCK_RULES: 만두류

## BLOCK_PRIORITY

R98_OUT_OF_SCOPE
R01_해외만두류
R02_딤섬
R03_교자
R04_일반만두
R97_만두류복합검토
R99_REVIEW

## 공통 처리 원칙

- BLOCK_PRIORITY를 위에서부터 적용한다.
- 특정 R block에 들어간 뒤에는 해당 block의 INTERNAL_PRIORITY를 위에서부터 적용한다.
- raw_generic_name은 2차 프롬프트에서 생성된 후보명이고, final_generic_name은 3차에서 확정하는 분석용 이름이다.
- final_generic_name 후보 목록은 “강제 통합 가능한 대표명 목록”이다.
- 해외만두류, 딤섬, 교자는 일반만두보다 먼저 판단한다.
- 샤오롱바오, 쇼마이, 하가우, 빠오즈, 모모, 페로기, 크랩랑군, 덤플링은 일반만두로 흡수하지 않고 R01_해외만두류에서 각각 별도 final_generic_name으로 보존한다.
- 딤섬은 특정 해외만두명이 명확하지 않을 때 final_generic_name = 딤섬으로 통합한다.
- 교자는 일반만두로 흡수하지 않고 final_generic_name = 교자로 통합한다.
- 군만두, 튀김만두, 야끼만두는 별도 final_generic_name으로 살리지 않고 일반만두의 final_generic_name = 만두로 통합한다.
- 위 특수 block에 해당하지 않는 일반 만두류는 final_generic_name = 만두로 통합한다.
- 만두류가 아닌 후보는 R98_OUT_OF_SCOPE로 보낸다.
- 완탕, 새우완탕, 사천 통새우 완탕처럼 국물형 완탕 메뉴는 만두류가 아니라 국,탕류로 이동한다.
- 만두류 안에는 남을 수 있지만 어떤 block인지 애매하면 R97_만두류복합검토로 보낸다.
- raw_generic_name에 정식, 세트, 코스 등 제공형태 신호가 남아 있더라도 만두류 본체가 명확하면 R98_OUT_OF_SCOPE로 보내지 않고 해당 만두 block에 배정한다.
- 만두류 여부 자체가 불명확하면 R99_REVIEW로 보낸다.
- AI가 새로운 assigned_block이나 family_group을 임의 생성하면 안 된다.
- AI가 CATEGORY_BLOCK_RULES에 없는 넓은 final_generic_name을 임의 생성하면 안 된다.
- 예시는 실제 데이터에 등장한 clean_text 기반으로만 작성한다.
- 데이터에 없는 가상 메뉴명은 예시로 쓰지 않는다.

## R98_OUT_OF_SCOPE

family_group = ""

목적:
만두류가 아니거나 다른 category가 더 적절한 후보를 분리한다.

INTERNAL_PRIORITY:
1. 국,탕류
2. 찌개,전골류
3. 면류
4. 밥류 또는 만두 외 식사 본체
5. 샐러드류
6. 소스,토핑,추가,옵션,비메뉴

매칭 조건:
- 완탕, 새우완탕, 사천 통새우 완탕처럼 국물형 완탕 메뉴
  → R98_OUT_OF_SCOPE

- 만둣국, 만두국, 떡만둣국, 떡만두국처럼 국,탕 본체가 강한 후보
  → R98_OUT_OF_SCOPE

- 만두전골, 만두탕처럼 찌개,전골,탕 본체가 강한 후보
  → R98_OUT_OF_SCOPE

- 만두라면, 만두칼국수, 칼국수만두, 사골손칼만두, 방태칼만두처럼 면 본체가 강한 후보
  → R98_OUT_OF_SCOPE

- 만두덮밥처럼 밥 본체가 강한 후보
  → R98_OUT_OF_SCOPE

- 정식, 세트, 코스 신호만 붙었고 만두류 본체가 명확한 후보는 R98_OUT_OF_SCOPE로 보내지 않는다.

- 군만두샐러드, 만두샐러드, 만두튀김샐러드처럼 샐러드 본체가 강한 후보
  → R98_OUT_OF_SCOPE

- 소스, 토핑, 추가, 옵션, 비메뉴 설명문
  → R98_OUT_OF_SCOPE

출력 규칙:
- assigned_block = R98_OUT_OF_SCOPE
- final_generic_name = raw_generic_name
- family_group = ""
- normalization_actions = ["exclude_or_correction"]
- review_status = correction_candidate

## R02_딤섬

family_group = 딤섬

final_generic_name 후보:
- 딤섬

INTERNAL_PRIORITY:
1. 딤섬

bucket 판단:
- 딤섬은 특정 해외만두명이 명확하지 않은 딤섬류를 처리하는 block이다.
- 샤오롱바오, 쇼마이, 하가우, 빠오즈가 명확하면 R01_해외만두류를 우선한다.
- 특정 이름 없이 딤섬 자체가 본체이면 final_generic_name = 딤섬으로 통합한다.

통합 규칙:
- 딤섬, 딤섬만두, 날치알새우딤섬, 돼지고기 딤섬, 마라딤섬, 마라크림딤섬, 블랙딤섬, 사이공 딤섬, 새우딤섬, 부추새우딤섬, 표고새우딤섬, 피아오량딤섬, 해물딤섬, 트러플딤섬
  → final_generic_name = 딤섬

주의 규칙:
- 샤오롱바오딤섬, 하가우 딤섬, 포시즌하가우 딤섬은 R01_해외만두류 우선이다.
- 딤섬세트, 딤섬코스처럼 제공형태 신호가 남아 있어도 딤섬 본체가 명확하면 R02_딤섬으로 배정하고 final_generic_name = 딤섬으로 통합한다.
- 세트, 코스 등 제공형태 신호는 attribute에 남긴다.


attribute 규칙:
- 새우, 고기, 돼지고기, 날치알, 마라, 크림, 블랙, 사이공, 표고, 해물, 트러플 등 세부 정보는 attribute에 남긴다.

## R04_일반만두

family_group = 일반만두

final_generic_name 후보:
- 만두

INTERNAL_PRIORITY:
1. 만두

bucket 판단:
- 일반만두는 최종 흡수 block이다.
- 해외만두류, 딤섬, 교자에 해당하지 않는 일반 만두류는 final_generic_name = 만두로 통합한다.
- 군만두, 튀김만두, 야끼만두는 별도 final_generic_name으로 살리지 않고 만두로 통합한다.
- 조리 방식이 찐/물/군/튀김/야끼/납작/비빔 등으로 달라도 메뉴 본체가 만두이면 final_generic_name = 만두로 통합한다.
- 모둠/모듬/반반/섞어 계열도 만두 구성형이면 final_generic_name = 만두로 통합하고, 구성 신호는 attribute에 남긴다.
- 미처 발견하지 못한 일반 만두명도 일반 만두로 판단 가능하면 final_generic_name = 만두로 통합한다.
- 다만 일반만두인지 해외만두류인지 불명확하면 R97 또는 R99로 보낸다.

주의 규칙:
- 샤오롱바오/쇼마이/하가우/빠오즈/Momo/Pierogies/페로기/크랩 랑군/핸드메이드덤플링 신호가 있으면 R01_해외만두류 우선이다.
- 딤섬 신호가 있으면 R02_딤섬 우선이다.
- 교자 신호가 있으면 R03_교자 우선이다.
- 완탕, 새우완탕, 사천 통새우 완탕은 R98_OUT_OF_SCOPE로 보낸다.
- 만둣국, 만두전골, 만두라면, 칼국수만두처럼 국/전골/면 본체가 강하면 R98_OUT_OF_SCOPE로 보낸다.
- 만두세트, 만두정식, 만두코스처럼 제공형태 신호가 남아 있어도 만두 본체가 명확하면 R04_일반만두로 배정하고 final_generic_name = 만두로 통합한다.
- 세트, 정식, 코스 등 제공형태 신호는 attribute에 남긴다.

attribute 규칙:
- 고기, 김치, 갈비, 고추, 왕, 손, 수제, 새우, 감자, 평양, 납작, 물, 찐, 군, 튀김, 야끼, 비빔, 유린, 홍유, 마라, 치즈, 명란, 능이, 메밀, 흑돼지, 반반, 모둠, 모듬, 섞어 등은 attribute에 남긴다.

## R97_만두류복합검토

family_group = 만두류복합검토

목적:
만두류 안에는 남을 수 있지만, 기존 R01~R04의 final_generic_name으로 자동 통합하면 의미 손실이나 category 혼동이 생길 수 있는 후보를 검토용으로 분리한다.

INTERNAL_PRIORITY:
1. 만두류 안에 남을 수 있지만 본체가 애매한 후보
2. 만두와 다른 메뉴 본체가 결합된 후보
3. 일반만두/딤섬/교자/해외만두류 중 판단이 애매한 후보
4. final_generic_name 후보로 자동 통합하면 의미 손실이 큰 후보

매칭 조건:
- 만두류는 맞지만 R01_해외만두류, R02_딤섬, R03_교자, R04_일반만두 중 어디로 보낼지 불명확한 후보
- 만두류 본체보다 다른 메뉴 구성, 식사형 구성, 사이드 구성이 강해서 자동 통합이 애매한 후보
- 만두튀김샐러드, 만두샐러드처럼 만두와 다른 메뉴 본체가 결합된 후보
- 만두류 안에 남을 가능성은 있지만 자동 통합하면 의미 손실이 큰 후보

제외 조건:
- 해외만두류 신호가 명확하면 R01
- 딤섬 신호가 명확하면 R02
- 교자 신호가 명확하면 R03
- 일반만두 신호가 명확하면 R04
- 만두류가 아님이 명확하면 R98
- 만두류 여부 자체가 불명확하면 R99

출력 규칙:
- assigned_block = R97_만두류복합검토
- final_generic_name = raw_generic_name
- family_group = 만두류복합검토
- normalization_actions = ["broad_merge_review"]
- review_status = needs_review

## R99_REVIEW

family_group = ""

목적:
만두류 여부, 음식명 여부, 또는 적용 가능한 R block 자체가 불명확한 후보를 최종 검토한다.

매칭 조건:
- 어떤 block에도 안정적으로 들어가지 않는 경우
- raw_generic_name만으로 만두류인지 알 수 없는 경우
- 음식명인지 옵션/소스/재료명인지 불명확한 경우
- 오타가 심해 만두류 여부 판단이 어려운 후보
- 브랜드명 또는 업장 고유명처럼 보이나 만두류 본체가 드러나지 않는 후보

출력 규칙:
- assigned_block = R99_REVIEW
- final_generic_name = raw_generic_name
- family_group = ""
- normalization_actions = ["broad_merge_review"]
- review_status = needs_review
