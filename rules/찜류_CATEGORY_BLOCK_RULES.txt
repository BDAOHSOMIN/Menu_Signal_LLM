# CATEGORY_BLOCK_RULES: 찜류

## BLOCK_PRIORITY

R98_OUT_OF_SCOPE  
R01_해외찜  
R02_사이드찜  
R03_육류찜  
R04_어패류찜  
R05_채소찜  
R97_찜류복합검토  
R99_REVIEW  


## 공통 처리 원칙
- 실제 예시는 찜류 데이터 예시로 프롬프트를 작성하였으나, 처리를 하면서 실제 찜류가 아닌 잘못된 내역이 있진 않은지 판단하면서 처리를 한다. 
- BLOCK_PRIORITY를 위에서부터 적용한다.
- 특정 R block에 들어간 뒤에는 해당 block의 INTERNAL_PRIORITY를 위에서부터 적용한다.
- raw_generic_name은 2차 프롬프트에서 생성된 후보명이고, final_generic_name은 3차에서 확정하는 분석용 이름이다.
- final_generic_name 후보 목록은 “강제 통합 가능한 대표명 목록”이다.
- raw_generic_name이 block 안의 final_generic_name 후보로 명확히 통합 가능하면 반드시 해당 final_generic_name으로 통합한다.
- final_generic_name 후보에 없더라도 raw_generic_name이 특정 R block/family_group에 명확히 속하면 새로운 넓은 대표명으로 억지 통합하지 않는다.
- 다만 R03_육류찜, R04_어패류찜, R05_채소찜은 흡수형 fallback을 가지므로, block 내부 규칙에 명시된 경우에는 각각 육류찜, 어패류찜, 채소찜으로 통합할 수 있다.
- R97_찜류복합검토와 R99_REVIEW는 검토용 block이므로 final_generic_name = raw_generic_name으로 유지한다.
- AI가 새로운 assigned_block이나 family_group을 임의 생성하면 안 된다.
- AI가 CATEGORY_BLOCK_RULES에 없는 넓은 final_generic_name을 임의 생성하면 안 된다.
- 찜류가 아닌 후보는 R98_OUT_OF_SCOPE로 보낸다.
- 해외찜, 사이드찜은 육류찜/어패류찜/채소찜보다 먼저 처리한다.
- 해외찜은 whitelist 보호 block이다.
- R01_해외찜 후보는 절대로 육류찜/어패류찜/채소찜/사이드찜으로 흡수하지 않는다.
- 해외요리처럼 보이더라도 R01 whitelist에 없으면 국내식 찜 block으로 임의 흡수하지 않는다.
- 찜류보다 조림류가 더 적절한 후보는 R98_OUT_OF_SCOPE로 보낸다.
- 찜류와 조림류 경계가 애매하면 R97_찜류복합검토로 보낸다.
- 계란찜, 꽃빵찜처럼 사이드 성격이 강한 찜은 R02_사이드찜에서 먼저 처리한다.
- 육류찜은 족발/보쌈/수육/편육/찜닭/등갈비찜/갈비찜/뼈찜/꼬리찜/편백찜/육류찜을 중심으로 처리한다.
- 등갈비찜은 갈비찜보다 먼저 판단한다.
- 보쌈수육, 수육보쌈처럼 보쌈과 수육이 결합된 후보는 보쌈을 우선한다.
- named dish에 해당하지 않지만 육류 재료 + 찜 신호가 명확하면 final_generic_name = 육류찜으로 흡수한다.
- 어패류찜은 아구찜/꽃게찜/코다리찜/대게찜/킹크랩찜/대구뽈찜/알곤이찜/랍스터찜/조개찜/굴·석화찜/전복찜/연체류찜/가오리찜/생선찜/해물찜/어패류찜을 중심으로 처리한다.
- 해물찜은 raw_generic_name이 해물찜 또는 해물찜 계열일 때만(실제로 해물찜이 들어가 있음) final_generic_name = 해물찜으로 처리한다.
- 위의 어패류 anchor에 명확히 걸리지 않지만 어류/패류/갑각류/연체류/해산물 재료 + 찜/통찜/술찜/스팀 신호가 있으면 final_generic_name = 어패류찜으로 통합한다.
- 채소찜은 채소/버섯/두부 등 비육류·비어패류 기반 찜류를 final_generic_name = 채소찜으로 통합한다.
- 회/숙회/물회/사시미/타다끼/카르파치오 등 회류 신호가 명확하면 찜류에 남기지 않고 R98_OUT_OF_SCOPE로 보낸다.
- 단독 해산물명만 있고 찜/통찜/술찜/보일링/스팀 신호가 없으면 R97 또는 R99로 보낸다.
- 난자완스는 찜류로 자동 승인하지 않고 R98_OUT_OF_SCOPE로 보낸다.
- 찜류 안에는 남을 수 있지만 자동 통합하면 의미 손실이나 category 혼동이 생기는 후보는 R97_찜류복합검토로 보낸다.
- 찜류 여부, 음식명 여부, 적용 가능한 block 자체가 불명확하면 R99_REVIEW로 보낸다.
- cuisine_signal은 category rule에서 적극적으로 추론하지 않는다. 명확한 해외 신호가 있어도 대부분 final_generic_name과 assigned_block으로 표현하고, cuisine_signal은 none으로 둬도 된다.


## R98_OUT_OF_SCOPE

family_group = ""

목적:  
찜류가 아니거나 다른 category가 더 적절한 후보를 분리한다.

매칭 조건:
- 회, 숙회, 물회, 사시미, 타다끼, 카르파치오처럼 회류 본체
- 문어숙회, 쭈꾸미숙회, 알배기쭈꾸미숙회, 데친오징어, 오징어데침, 낙지데침, 쭈꾸미데침처럼 숙회/데침 본체
- 샤브샤브, 전골, 탕, 국, 찌개, 라면, 칼국수처럼 국물/탕/면 본체
- 볶음, 구이, 튀김, 전, 부침, 무침처럼 찜류가 아닌 조리법 본체
- 조림류가 더 적절한 후보
- 덮밥, 정식, 세트, 코스, 샐러드, 쌈, 롤처럼 다른 메뉴 본체가 강한 후보
- 닭도리탕, 닭볶음탕처럼 탕/볶음탕 본체
- 난자완스, 새우난자완스, 치즈난자완스처럼 찜류 anchor가 아닌 중식 완자/소스 요리
- 부타가쿠니, 부르기뇽, 타진, 캐서롤, 오향장육, 산니백육, 수주육편, 커우러우처럼 이번 R01_해외찜 whitelist에서 제외한 해외 고유요리 중 조림/브레이즈/스튜/냉채/소스요리 성격이 더 강한 후보
- 전가복, 불도장처럼 찜류 자동 처리가 위험하고 중식 요리/탕·스프·코스요리 성격이 강한 후보
- 소스명, 토핑명, 추가, 옵션, 비메뉴 설명문

출력 규칙:
- assigned_block = R98_OUT_OF_SCOPE
- final_generic_name = raw_generic_name
- family_group = ""
- normalization_actions = ["exclude_or_correction"]
- review_status = correction_candidate


## R01_해외찜

family_group = 해외찜

final_generic_name 후보:
- 보일링씨푸드
- 차완무시
- 세이로무시
- 샥스핀찜
- 동파육
- 오소부코

INTERNAL_PRIORITY:
1. 보일링씨푸드
2. 차완무시
3. 세이로무시
4. 샥스핀찜
5. 동파육
6. 오소부코

bucket 판단:
- 해외찜은 한국식 육류찜, 어패류찜, 사이드찜으로 흡수하면 의미가 사라지는 고유 해외 찜/스팀/보일링/브레이즈 메뉴를 보호하는 block이다.
- R01_해외찜은 whitelist 방식으로만 적용한다.
- 아래 final_generic_name 후보에 명확히 해당하는 경우에만 R01_해외찜으로 처리한다.
- 해외 음식명처럼 보이더라도 아래 후보에 없으면 육류찜/어패류찜/채소찜으로 흡수하지 않는다.
- 보일링씨푸드는 해물찜/어패류찜/랍스터찜으로 보내지 않는다.
- 차완무시는 계란찜으로 보내지 않는다.
- 세이로무시는 편백찜/육류찜/채소찜으로 보내지 않는다.
- 샥스핀찜은 해물찜/어패류찜으로 보내지 않는다.
- 동파육, 오소부코는 국내식 육류찜 fallback으로 흡수하지 않고 각각 고유 final_generic_name으로 보존한다.
- 부타가쿠니, 부르기뇽, 타진, 캐서롤, 오향장육, 산니백육, 수주육편, 커우러우는 R01_해외찜에서 제외한다.
- 제외된 해외 고유요리는 R03_육류찜이나 R04_어패류찜으로 임의 흡수하지 않는다.
- 제외된 해외 고유요리가 조림류에 더 적절하면 R98_OUT_OF_SCOPE로 보낸다.
- 제외된 해외 고유요리가 찜류 안에 남을 가능성은 있으나 애매하면 R97_찜류복합검토로 보낸다.

통합 규칙:
- 보일링씨푸드, 보일링 씨푸드, 루이지애나 보일링씨푸드, 선셋 보일링씨푸드, 보일링 씨푸드 대, 보일링 씨푸드 중, 보일링 씨푸드 소, 보일링 씨푸드 특대, boiling seafood, seafood boil
  → final_generic_name = 보일링씨푸드

- 그린홍합 보일링, 랍스터테일 보일링, 보일링 랍스터, 보일링 새우, 보일링쉬림프, 보일링쉬림프 M, 보일링쉬림프 L, 크레이피쉬 보일링
  → final_generic_name = 보일링씨푸드

- 차완무시, 자완무시, 일본식계란찜, 마파 차완무시, 우니 트러플 게살 차완무시, 트러플 차완무시
  → final_generic_name = 차완무시

- 세이로무시, 돼지고기 세이로무시, 세이로무시찜
  → final_generic_name = 세이로무시

- 샥스핀찜, 상해식 샥스핀찜, 송이전복통샥스핀찜, 통샥스핀찜, 홍쇼샥스핀, 발채샥스핀, 상어지느러미찜
  → final_generic_name = 샥스핀찜

- 동파육, 동파육찜, 동파육조림, 마라동파육, 오향동파육, 통삼겹 동파육, 항정살 동파육, 동파육 with Buns
  → final_generic_name = 동파육

- 오소부코, osso buco, ossobuco
  → final_generic_name = 오소부코

주의 규칙:
- 해외찜은 R02_사이드찜, R03_육류찜, R04_어패류찜, R05_채소찜보다 먼저 적용한다.
- R01 whitelist 후보는 절대로 국내식 찜 block으로 흡수하지 않는다.
- 보일링 랍스터는 랍스터찜이 아니라 보일링씨푸드이다.
- 보일링 새우는 어패류찜이 아니라 보일링씨푸드이다.
- 차완무시는 계란찜이 아니라 차완무시이다.
- 세이로무시는 편백찜이 아니라 세이로무시이다.
- 샥스핀찜은 어패류찜이 아니라 샥스핀찜이다.
- 동파육은 수육/보쌈/육류찜이 아니라 동파육이다.
- 오소부코는 육류찜 fallback이 아니라 오소부코이다.
- 부타가쿠니, 부르기뇽, 타진, 캐서롤, 오향장육, 산니백육, 수주육편, 커우러우는 R01_해외찜으로 처리하지 않는다.
- 해외 고유 메뉴처럼 보이지만 R01 후보에 없는 경우는 국내식 찜 block으로 흡수하지 않고 R97 또는 R98로 보낸다.

attribute 규칙:
- 해산물, 홍합, 랍스터, 새우, 쉬림프, 크레이피쉬, 돼지고기, 소고기, 계란, 샥스핀, 트러플, 우니, 중식, 일식, 이탈리아식 등 세부 재료와 조리 신호는 attribute에 남긴다.


## R02_사이드찜

family_group = 사이드찜

final_generic_name 후보:
- 계란찜
- 꽃빵찜

INTERNAL_PRIORITY:
1. 계란찜
2. 꽃빵찜

bucket 판단:
- 사이드찜은 메인 찜요리보다 사이드 반찬/곁들임 성격이 강한 찜 후보를 처리한다.
- 계란찜은 다른 육류찜/어패류찜보다 먼저 R02_사이드찜에서 처리한다.
- 꽃빵찜은 final_generic_name = 꽃빵찜으로 보존한다.
- 차완무시는 계란찜과 유사하지만 R01_해외찜으로 보낸다.

통합 규칙:
- 계란찜, 달걀찜, 치즈계란찜, 폭탄계란찜, 뚝배기계란찜, 푸딩계란찜, 계란찜추가
  → final_generic_name = 계란찜

- 꽃빵찜, 꽃빵
  → final_generic_name = 꽃빵찜

주의 규칙:
- 차완무시, 일본식계란찜은 R01_해외찜 우선이다.
- 낙지계란찜, 굴계란찜처럼 재료가 붙어도 계란찜이 본체이면 계란찜으로 처리한다.
- 계란찜이지만 조림/탕/국 형태가 본체이면 R98_OUT_OF_SCOPE로 보낸다.

attribute 규칙:
- 치즈, 폭탄, 뚝배기, 푸딩, 추가, 낙지, 굴 등 세부 정보는 attribute에 남긴다.


## R03_육류찜

family_group = 육류찜

final_generic_name 후보:
- 족발
- 보쌈
- 수육
- 편육
- 찜닭
- 등갈비찜
- 갈비찜
- 뼈찜
- 꼬리찜
- 편백찜
- 육류찜

INTERNAL_PRIORITY:
1. 족발
2. 보쌈
3. 수육
4. 편육
5. 찜닭
6. 등갈비찜
7. 갈비찜
8. 뼈찜
9. 꼬리찜
10. 편백찜
11. 육류찜

bucket 판단:
- 육류찜은 육류/고기/부속 재료가 본체인 찜류를 처리한다.
- 족발은 수식어가 있어도 final_generic_name = 족발로 통합한다.
- 보쌈수육처럼 보쌈과 수육이 결합된 후보는 final_generic_name = 보쌈으로 처리한다.
- 수육은 보쌈 신호가 없고 수육이 본체일 때 final_generic_name = 수육으로 처리한다.
- 편육은 수육으로 합치지 않고 final_generic_name = 편육으로 보존한다.
- 찜닭은 닭찜/닭도리탕과 구분하여 final_generic_name = 찜닭으로 처리한다.
- 등갈비찜은 갈비찜보다 먼저 처리한다.
- 쪽갈비찜, 묵은지쪽갈비찜, 묵은지등갈비찜, 등갈비묵은지찜은 등갈비찜으로 처리한다.

INTERNAL_PRIORITY:
1. 육류찜 + 어패류찜 복합
2. 해외찜 whitelist에 없지만 찜류 안에 남을 가능성이 있는 해외 고유요리
3. 찜류와 조림류 경계가 애매한 후보
4. 단독 재료명이라 찜 여부가 불명확한 후보
5. 찜류 안에 남을 수 있지만 조리 형태가 애매한 후보
6. final_generic_name 후보로 자동 통합하면 의미 손실이 큰 후보

매칭 조건:
- 등갈비해물찜, 낙지전복조개갈비찜, 문어전복조개갈비찜, 오징어 전복 조개갈비찜처럼 육류찜과 어패류찜이 강하게 결합된 후보
- 돌문어, 통문어, 석화, 꽃게, 완도전복, 랍스터 테일처럼 단독 재료명만 있고 찜 여부가 불명확한 후보
- 찜/통찜/술찜/보일링/스팀 신호는 있으나 어떤 final_generic_name으로 보낼지 불명확한 후보
- 해외 고유 메뉴처럼 보이지만 R01_해외찜 whitelist에 없는 후보 중 찜류 안에 남을 가능성이 있는 후보
- 찜류인지 조림류인지 불명확한 후보
- 찜류 안에 남을 가능성은 있지만 자동 통합하면 의미 손실이 큰 후보

제외 조건:
- R01_해외찜 whitelist 후보가 명확하면 R01
- 사이드찜 신호가 명확하면 R02
- 국내식 육류찜 신호가 명확하면 R03
- 국내식 어패류찜 신호가 명확하면 R04
- 국내식 채소찜 신호가 명확하면 R05
- 찜류가 아님이 명확하거나 조림류가 더 적절하면 R98
- 찜류 여부 자체가 불명확하면 R99

출력 규칙:
- assigned_block = R97_찜류복합검토
- final_generic_name = raw_generic_name
- family_group = 찜류복합검토
- normalization_actions = ["broad_merge_review"]
- review_status = needs_review


## R99_REVIEW

family_group = ""

목적:  
찜류 여부, 음식명 여부, 또는 적용 가능한 R block 자체가 불명확한 후보를 최종 검토한다.

매칭 조건:
- 어떤 block에도 안정적으로 들어가지 않는 경우
- raw_generic_name만으로 찜류인지 알 수 없는 경우
- 음식명인지 옵션/소스/재료명인지 불명확한 경우
- 오타가 심해 찜류 여부 판단이 어려운 후보
- 브랜드명 또는 업장 고유명처럼 보이나 찜류 본체가 드러나지 않는 후보
- 재료가 불명확하거나 육류/어패류/채소 중 어디에도 확실히 속하지 않는 후보
- 찜류인지 조림류인지 판단하기 어려운 후보

출력 규칙:
- assigned_block = R99_REVIEW
- final_generic_name = raw_generic_name
- family_group = ""
- normalization_actions = ["broad_merge_review"]
- review_status = needs_review
