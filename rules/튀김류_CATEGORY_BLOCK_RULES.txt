# CATEGORY_BLOCK_RULES: 튀김류

R98_OUT_OF_SCOPE
R01_일식튀김류
R02_중식튀김류
R03_동남아튀김류
R04_양식튀김류
R05_고로케류
R06_까스카츠류
R07_닭튀김류
R08_새우튀김류
R09_감자프라이즈류

## 공통 처리 원칙

- BLOCK_PRIORITY를 위에서부터 적용한다.
- 특정 R block에 들어간 뒤에는 해당 block의 INTERNAL_PRIORITY를 위에서부터 적용한다.
- raw_generic_name은 2차 프롬프트에서 생성된 후보명이고, final_generic_name은 3차에서 확정하는 분석용 이름이다.
- final_generic_name 후보 목록은 “강제 통합 가능한 대표명 목록”이다.
- raw_generic_name이 block 안의 final_generic_name 후보로 명확히 통합 가능하면 반드시 해당 final_generic_name으로 통합한다.
- final_generic_name 후보에 없더라도 해당 block의 fallback final_generic_name이 존재하면 fallback final_generic_name으로 통합한다.
  - 예: 에비후라이 → R01_일식튀김류 / final_generic_name = 일식튀김
  - 예: 깐벤오징어 → R02_중식튀김류 / final_generic_name = 중식튀김
  - 예: 슈니첼 → R04_양식튀김류 / final_generic_name = 양식튀김
  - 예: 오징어튀김 → R10_일반튀김류 / final_generic_name = 튀김
- R97_튀김류복합검토와 R99_REVIEW는 검토용 block이므로 final_generic_name = raw_generic_name으로 유지한다.
- AI가 새로운 assigned_block이나 family_group을 임의 생성하면 안 된다.
- AI가 CATEGORY_BLOCK_RULES에 없는 final_generic_name을 임의 생성하면 안 된다.
- 튀김류가 아닌 후보는 R98_OUT_OF_SCOPE로 보낸다.
- 단, 스프링롤/짜조처럼 동남아 튀김 고유 음식명 신호가 명확하면 R98로 보내지 않고 R03_동남아튀김류를 우선한다.
- 샐러드롤/비튀김 롤 본체가 강한 경우만 R98_OUT_OF_SCOPE로 보낸다.

- 단, 메뉴명에 정식/안주/모듬/모둠/세트/반반이 포함되어도 튀김 본체가 명확하면 R98로 보내지 않는다.
  - 실제 예: 돈까스정식 → R06_까스카츠류 / 까스
  - 실제 예: 등심카츠 정식 → R06_까스카츠류 / 카츠
  - 실제 예: 돈까스안주 → R06_까스카츠류 / 까스
  - 실제 예: 모듬튀김 → R10_일반튀김류 / 튀김
  - 실제 예: 모둠튀김 → R10_일반튀김류 / 튀김
  - 실제 예: 해물모듬튀김 → R10_일반튀김류 / 튀김
  - 실제 예: 덴푸라 모둠 → R01_일식튀김류 / 덴푸라
  - 실제 예: 모듬 덴푸라 → R01_일식튀김류 / 덴푸라
  - 실제 예: 모듬 가라아게 → R01_일식튀김류 / 가라아게
  
- 반대로 샐러드/롤/버거/떡볶이/카레/나베 등 다른 메뉴 본체가 더 강하면 R98_OUT_OF_SCOPE 또는 R97_튀김류복합검토로 보낸다.
  - 실제 예: 새우튀김샐러드 → R98_OUT_OF_SCOPE
  - 실제 예: 샐러드돈까스 → R98_OUT_OF_SCOPE
  - 실제 예: 돈카츠 샐러드 롤 → R98_OUT_OF_SCOPE
  - 실제 예: 싸이버거 → R98_OUT_OF_SCOPE
  - 실제 예: 눈꽃치즈모짜렐라 돈까스 떡볶이 → R98_OUT_OF_SCOPE

- 해외 고유 튀김류는 재료 기반 튀김류보다 먼저 처리한다.
- 일식튀김류, 중식튀김류, 동남아튀김류, 양식튀김류는 닭튀김류, 새우튀김류, 감자프라이즈류, 일반튀김류보다 먼저 처리한다.
- 중식튀김류는 강하게 판단한다. 중식 소스/조리명 신호(깐풍,깐벤 등)가 있으면 재료보다 R02_중식튀김류를 우선한다.
  - 실제 예: 깐풍가지, 깐풍동고, 깐풍오징어, 깐풍새우 → 깐풍튀김
  - 실제 예: 어향동고, 어향가지튀김, 가지탕수, 두부탕수 → 중식튀김
  
- 일식튀김류는 일본식 고유명 신호가 있으면 재료보다 R01_일식튀김류를 우선한다.
  - 실제 예: 에비텐, 에비덴푸라, 새우덴푸라 → 덴푸라
  - 실제 예: 에비후라이, 카키후라이, 게소아게, 쿠시카츠, 테바사키 → 일식튀김
  
- 고로케류는 감자튀김류나 일반튀김류로 흡수하지 않고 별도 block에서 먼저 처리한다.

- 까스카츠류는 닭튀김류, 새우튀김류, 일반튀김류보다 먼저 처리한다.
  - 실제 예: 치킨까스 → 까스
  - 실제 예: 치킨카츠 → 카츠
  - 실제 예: 생선까스 → 까스
  - 실제 예: 생선카츠 → 카츠
  - 실제 예: 히레까스 → 카츠
  
- 새우튀김류는 새우 신호가 확실하고, 중식/일식/동남아 고유 튀김 신호가 없을 때만 처리한다.
  - 실제 예: 새우튀김, 왕새우튀김, 대하튀김, 통새우튀김 → 새우튀김
  - 실제 예: 깐쇼새우, 크림새우, 칠리새우, 멘보샤 → R02_중식튀김류
  - 실제 예: 에비텐, 에비후라이 → R01_일식튀김류
  - 실제 예: 텃만꿍 → R03_동남아튀김류
  
- 감자프라이즈류는 감자 기반 튀김/프라이즈/포테이토 사이드 메뉴를 처리한다.

- R10_일반튀김류는 앞선 block에 해당하지 않지만 튀김류로 명확한 후보를 모두 final_generic_name = 튀김으로 통합한다.
  - 실제 예: 오징어튀김, 가지튀김, 고추튀김, 고구마튀김, 굴튀김, 복튀김, 두부튀김, 김말이튀김, 모듬튀김 → 튀김
- 재료, 소스, 부위, 스타일 정보는 final_generic_name으로 과하게 쪼개지 말고 attribute에 남긴다.

## R98_OUT_OF_SCOPE

family_group = ""

목적:
튀김류가 아니거나, 튀김이 포함되어도 다른 category 본체가 더 강한 후보를 분리한다.

매칭 조건:
- 구이, 찜, 볶음, 조림, 탕, 국, 찌개, 전골, 무침, 회, 숙회, 전, 부침처럼 튀김류가 아닌 조리법 본체
- 밥, 덮밥, 면, 국수, 우동, 파스타, 샐러드, 버거, 샌드위치, 피자, 떡볶이처럼 다른 메뉴 본체가 강한 후보
- 소스명, 드레싱명, 토핑명, 추가, 옵션, 비메뉴 설명문

R98로 보내는 실제 예:
- 새우튀김샐러드
- 샐러드돈까스
- 돈카츠 샐러드 롤
- 눈꽃치즈모짜렐라 돈까스 떡볶이 -> 떡볶이로 포함될 수 있을 듯 함.

R98로 보내지 않는 실제 예:
- 돈까스정식 → R06_까스카츠류 / 까스
- 등심카츠 정식 → R06_까스카츠류 / 카츠
- 돈까스안주 → R06_까스카츠류 / 까스
- 모듬튀김 → R10_일반튀김류 / 튀김
- 모둠튀김 → R10_일반튀김류 / 튀김
- 모듬 덴푸라 → R01_일식튀김류 / 덴푸라

출력 규칙:
- assigned_block = R98_OUT_OF_SCOPE
- final_generic_name = raw_generic_name
- family_group = ""
- normalization_actions = ["exclude_or_correction"]
- review_status = correction_candidate


## R01_일식튀김류

family_group = 일식튀김류

final_generic_name 후보:
- 가라아게
- 덴푸라
- 치킨난반
- 일식튀김

INTERNAL_PRIORITY:
1. 치킨난반
2. 덴푸라
3. 가라아게
4. 일식튀김

bucket 판단:
- 일식튀김류는 일본식 고유 튀김 메뉴를 처리하는 block이다.
- 일본식 고유명 신호가 명확하면 재료 기반 block보다 R01_일식튀김류를 우선한다.
- 가라아게는 닭튀김/치킨으로 보내지 않고 final_generic_name = 가라아게로 보존한다.
- 덴푸라/덴뿌라/텐푸라/텐뿌라 계열은 final_generic_name = 덴푸라로 통합한다.
- 에비텐, 에비덴푸라, 새우덴푸라, 덴푸라 모둠, 모듬 덴푸라는 final_generic_name = 덴푸라로 통합한다.
- 치킨난반/치킨 남방/치킨 난반은 final_generic_name = 치킨난반으로 통합한다.
- 쿠시카츠, 테바사키, 가키아게, 게소아게, 이카게소, 카키후라이, 아지후라이, 에비후라이처럼 일식 튀김 신호는 강하지만 별도 final로 살리지 않는 후보는 final_generic_name = 일식튀김으로 통합한다.

통합 규칙:
- 가라아게, 치킨가라아게, 갈비치킨 가라아게, 모듬 가라아게
  → final_generic_name = 가라아게

- 덴푸라, 덴뿌라, 에비텐, 에비덴푸라, 새우덴푸라, 덴푸라 모둠, 모듬 덴푸라, 모듬덴푸라모리아와세
  → final_generic_name = 덴푸라

- 치킨난반, 치킨 난반, 계란 타르타르 치킨난반
  → final_generic_name = 치킨난반

- 쿠시카츠, 모듬쿠시카츠, 모듬쿠시까츠, 테바사키, 가키아게, 게소아게, 이카게소, 카키후라이, 아지후라이, 에비후라이
  → final_generic_name = 일식튀김

주의 규칙:
- 에비텐/에비덴푸라/새우덴푸라는 덴푸라로 처리한다.
- 에비후라이/에비프라이/에비튀김은 새우튀김이 아니라 일식튀김으로 처리한다.
- 새우후라이/왕새우후라이는 에비같은 일시 튀김의 신호가 없으면 R08_새우튀김류로 보낸다.
- 쿠시카츠는 카츠가 포함되어도 R06_까스카츠류가 아니라 R01_일식튀김류에서 처리한다. (실제 쿠시카츠가 카츠인지 튀김꼬치인지 웹지식으로 판단한다.)
- 테바사키는 윙봉이나 닭튀김이 아니라 일식튀김으로 처리한다.
- 돈카츠/히레카츠/로스카츠는 R06_까스카츠류로 보낸다.

attribute 규칙:
- 닭, 치킨, 새우, 에비, 야채, 모듬, 꼬치, 게소, 굴, 후라이, 난반 등 세부 정보는 attribute에 남긴다.


## R02_중식튀김류

family_group = 중식튀김류

final_generic_name 후보:
- 탕수육
- 꿔바로우
- 깐풍튀김
- 유린기
- 멘보샤
- 깐쇼새우
- 크림새우
- 칠리새우
- 춘권
- 중식튀김

INTERNAL_PRIORITY:
1. 꿔바로우
2. 탕수육
3. 깐풍튀김
4. 유린기
5. 멘보샤
6. 깐쇼새우
7. 크림새우
8. 칠리새우
9. 춘권
10. 중식튀김

bucket 판단:
- 중식튀김류는 중식 소스/조리명과 결합된 튀김 메뉴를 처리한다.
- 중식 신호가 명확하면 재료 기반 block보다 R02_중식튀김류를 우선한다.
- 탕수육과 꿔바로우는 분리한다.
- 깐풍기/깐풍육/깐풍새우처럼 재료가 달라도 깐풍 조리 신호가 명확하면 final_generic_name = 깐풍튀김으로 통합한다.
- 유린기/유린치킨 계열은 final_generic_name = 유린기으로 통합한다.
- 깐쇼새우/크림새우/칠리새우는 새우튀김류보다 중식튀김류를 우선한다.
- 멘보샤는 새우튀김류가 아니라 중식튀김류에서 처리한다.
- 가자미탕수 같은 탕수 라인은 중식튀김안에 넣는다.
- 후보 final_generic_name으로 확정하기 어렵지만 중식 소스 튀김 신호가 명확하면 final_generic_name = 중식튀김으로 통합한다.

통합 규칙:
- 탕수육, 찹쌀탕수육, 미니탕수육, 사천탕수육, 버섯탕수육, 목화솜탕수육, 누룽지 고추탕수육, 돼지고기 탕수육, 생등심 탕수육, 볶음탕수육
  → final_generic_name = 탕수육

- 꿔바로우, 궈바로우, 궈버로우, 궈보러우, 겉바속촉 꿔바로우, 겉바속촉 동북 꿔바로우, 구름꿔바로우
  → final_generic_name = 꿔바로우

- 깐풍기, 간풍기, 깐풍육, 간풍육, 깐풍새우, 간풍새우, 깐풍중새우, 간풍중새우, 깐풍동고, 깐풍가지, 가지깐풍기, 깐풍오징어, 깐풍대하, 깐풍치킨, 사천깐풍기, 황비홍깐풍기, 칠리깐풍새우, 모듬깐풍
  → final_generic_name = 깐풍튀김

- 유린기, 유린치킨, 고추유린기, 고추바삭유린기, 고추듬뿍유린기
  → final_generic_name = 유린기

- 멘보샤, 마카롱멘보샤, 바게트 멘보샤, 가지멘보샤, 롤멘보샤
  → final_generic_name = 멘보샤

- 튀김류 안에 남을 가능성은 있지만 자동 통합하면 의미 손실이 큰 후보

## R99_REVIEW

목적:

- 어떤 block에도 안정적으로 들어가지 않는 경우
- raw_generic_name만으로 튀김류인지 알 수 없는 경우

- 재료가 불명확하거나 어느 튀김 축에도 확실히 속하지 않는 후보

- final_generic_name = raw_generic_name
- family_group = ""

- review_status = needs_review
