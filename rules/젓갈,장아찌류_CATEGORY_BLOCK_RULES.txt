# CATEGORY_BLOCK_RULES: 젓갈/장아찌류

## BLOCK_PRIORITY

R98_OUT_OF_SCOPE
R01_게장류
R02_새우장류
R03_식해류
R04_해외특수염장류
R05_장아찌/절임류
R06_젓갈류
R97_젓갈장아찌류복합검토
R99_REVIEW


## 공통 처리 원칙

- BLOCK_PRIORITY를 위에서부터 적용한다.
- 특정 R block에 들어간 뒤에는 해당 block의 INTERNAL_PRIORITY를 위에서부터 적용한다.
- raw_generic_name은 2차 프롬프트에서 생성된 후보명이고, final_generic_name은 3차에서 확정하는 분석용 이름이다.
- final_generic_name 후보 목록은 “강제 통합 가능한 대표명 목록”이다.
- raw_generic_name이 block 안의 final_generic_name 후보로 명확히 통합 가능하면 반드시 해당 final_generic_name으로 통합한다.
- final_generic_name 후보에 없더라도, raw_generic_name이 특정 R block/family_group에 명확히 속하면 새로운 넓은 대표명으로 억지 통합하지 않는다.
- 이 경우 final_generic_name = raw_generic_name으로 유지하고, family_group은 해당 R block의 family_group으로 출력하며, normalization_actions = ["broad_merge_review"], review_status = needs_review로 둔다.
- R97_젓갈장아찌류복합검토와 R99_REVIEW는 검토용 block이므로 final_generic_name = raw_generic_name으로 유지한다.
- AI가 새로운 assigned_block이나 family_group을 임의 생성하면 안 된다.
- AI가 CATEGORY_BLOCK_RULES에 없는 넓은 final_generic_name을 임의 생성하면 안 된다.
- 젓갈/장아찌류가 아닌 후보는 R98_OUT_OF_SCOPE로 보낸다.
- 게장류, 새우장류, 식해류, 해외특수염장류, 장아찌/절임류는 젓갈류보다 먼저 처리한다.
- 젓/젖/젓갈/명란/어리굴젓 등 젓갈 신호가 명확하면 세부 이름을 살리지 않고 final_generic_name = 젓갈로 통합한다.
- 게장류는 간장/양념/꽃게/참게/황게 등 세부 차이를 나누지 않고 final_generic_name = 게장으로 통합한다.
- 새우장류는 간장/양념/수제/통새우/왕새우 등 세부 차이를 나누지 않고 final_generic_name = 새우장으로 통합한다.
- 식해류는 가자미/갈치/오징어 등 재료 차이를 나누지 않고 final_generic_name = 식해로 통합한다.
- 해외특수염장류는 츠케모노, 타코와사비, 고노와다, 보타르가, 캐비어, 엔초비를 별도 축으로 직접 처리한다.
- 츠케모노, 타코와사비, 고노와다, 보타르가, 캐비어, 엔초비는 젓갈, 장아찌, 절임, 피클로 뭉개지 않는다.
- 고노와다, 보타르가, 캐비어, 엔초비, 타코와사비는 R97 검토로 보내지 않고 R04_해외특수염장류에서 직접 final_generic_name을 확정한다.
- 장아찌/절임류는 장아찌, 절임, 피클 정도만 구분한다.
- 연어장, 전복장, 생참치장, 꼬막장처럼 별도 축으로 자동 통합하기 애매한 해산물장 후보는 R97_젓갈장아찌류복합검토로 보낸다.
- 나머지 후보 중 젓갈/장아찌류 안에 남을 수 있지만 기존 R01~R06으로 확정하기 어려운 후보는 R97_젓갈장아찌류복합검토로 보낸다.
- 젓갈/장아찌류 여부, 음식명 여부, 적용 가능한 block 자체가 불명확하면 R99_REVIEW로 보낸다.
- 해당 category의 cuisine_signal은 category rule에서 적극적으로 추론하지 않는다. 대부분 cuisine_signal = none으로 둬도 된다.
- 단, 음식명 자체가 명확한 문화권 신호를 갖는 경우에만 보조적으로 남길 수 있다. 예: 츠케모노/타코와사비/고노와다 = 일식, 보타르가/캐비어/엔초비 = 유럽식 가능.


## R98_OUT_OF_SCOPE

family_group = ""

목적:
젓갈/장아찌류가 아니거나 다른 category가 더 적절한 후보를 분리한다.

매칭 조건:
- 김치, 갓김치, 굴김치, 쪽파김치, 묵은지, 보쌈김치속처럼 김치류 본체
- 석화, 굴석화, 삼배체굴, 거북손, 소라, 고동, 꼬막데침, 꼬막한접시처럼 단품 해산물/데침/한접시 본체
- 건어물, 먹태, 먹태포, 쥐포, 노가리, 과메기, 오다리처럼 건어물/마른안주 본체
- 무침, 꼬막무침, 꽃게무침처럼 무침류 본체가 명확한 후보. 단, 게장류로 통용되는 양념꽃게장/꽃게장 후보는 R01_게장류로 보낸다. 
- 또한, 장아찌무침처럼 장아찌 본체가 명확한 후보는 R05_장아찌/절임류로 보낸다.
- 쌈장, 마요, 소스, 막장, 강된장처럼 소스/장류 본체가 강한 후보
- 군함, 롤, 초밥, 스시, 카펠리니, 파스타, 까나페처럼 다른 메뉴 형태가 본체인 후보
- 엔초비 파스타, 엔초비 카펠리니, 캐비어 초밥, 캐비어 군함, 캐비어 롤, 타코와사비 군함처럼 해외특수염장류 재료가 들어가지만 다른 메뉴 형태가 본체인 후보
- 추가, 옵션, 세트, 코스, 비메뉴 설명문

출력 규칙:
- assigned_block = R98_OUT_OF_SCOPE
- final_generic_name = raw_generic_name
- family_group = ""
- normalization_actions = ["exclude_or_correction"]
- review_status = correction_candidate


## R01_게장류

family_group = 게장류

final_generic_name 후보:
- 게장

INTERNAL_PRIORITY:
1. 게장

bucket 판단:
- 게장류는 간장게장/양념게장/꽃게장/참게장/황게장 등 게를 간장 또는 양념에 절인 메뉴를 처리한다.
- 간장/양념/참게/황게/돌게/꽃게/정식/밥상 등의 차이는 final_generic_name으로 나누지 않고 attribute에 남긴다.
- 게장 신호가 명확하면 final_generic_name = 게장으로 통합한다.
- 게장류로 보이나 게장인지 단품게살/다른 해산물장인지 애매하면 R97_젓갈장아찌류복합검토로 보낸다.

통합 규칙:
- 게장, 간장게장, 양념게장, 꽃게장, 양념꽃게장, 참게간장게장, 참게장, 황게장, 칠게장, 돌게장, 제주황게장, 서해안간장게장, 명품 간장 게장, 간장 꽃게
  → final_generic_name = 게장

- 게장정식, 간장게장정식, 게장밥상, 간장게장밥상, 꽃게장백반, 꽃게장정식, 돌게장백반, 돌게장정식
  → final_generic_name = 게장

주의 규칙:
- 게장정식/게장밥상은 메뉴 본체가 게장이면 게장으로 통합하고, 정식/밥상은 attribute에 남긴다.
- 게살군함, 게살초밥, 게살롤처럼 다른 메뉴 형태가 본체이면 R98_OUT_OF_SCOPE로 보낸다.

attribute 규칙:
- 간장, 양념, 꽃게, 참게, 황게, 칠게, 돌게, 정식, 밥상, 백반, 서해안, 제주, 명품 등은 attribute에 남긴다.

- 날치알새우, 치즈새우롤처럼 새우장류가 아니면 R98_OUT_OF_SCOPE 또는 R97로 보낸다.

## R04_해외특수염장류

family_group = 해외특수염장류

- 츠케모노
- 타코와사비
- 고노와다
- 보타르가
- 캐비어
- 엔초비

INTERNAL_PRIORITY:
1. 타코와사비
2. 고노와다
3. 보타르가
4. 캐비어
5. 엔초비
6. 츠케모노

bucket 판단:
- 해외특수염장류는 일식/해외식 염장, 절임, 발효, 숙성 해산물 또는 고급 염장 식재료 계열 후보를 처리한다.
- 츠케모노는 일식 절임류이지만 일반 장아찌/절임류로 보내지 않고 final_generic_name = 츠케모노로 통합한다.
- 타코와사비는 문어/낙지 계열 와사비 절임 안주로 보고 final_generic_name = 타코와사비로 통합한다.
- 고노와다는 해삼 내장 염장/발효류로 보고 final_generic_name = 고노와다로 통합한다.
- 보타르가는 어란 염장/건조 식재료로 보고 final_generic_name = 보타르가로 통합한다.
- 캐비어/캐비아는 철갑상어알 또는 고급 어란류 신호로 보고 final_generic_name = 캐비어로 통합한다.
- 엔초비/엔쵸비/안초비는 염장 멸치류 신호로 보고 final_generic_name = 엔초비로 통합한다.
- 위 항목들은 젓갈, 장아찌, 절임, 피클로 뭉개지 않는다.
- 단, 다른 메뉴 형태가 본체인 경우에는 해외특수염장류로 처리하지 않는다.

통합 규칙:

  → final_generic_name = 츠케모노

- 타코와사비, 타코 와사비, 타코와사비젓, 문어와사비, 문어 와사비, 낙지와사비, 낙지 와사비
  → final_generic_name = 타코와사비

- 고노와다, 코노와다, 고노와다젓, 시로메고노와다, 시로미고노와다
  → final_generic_name = 고노와다

- 보타르가, Bottarga, bottarga, 사르데냐 보타르가, 어란 보타르가
  → final_generic_name = 보타르가

- 캐비어, 캐비아, caviar, Caviar, 캐비아리 캐비어, 철갑상어 캐비어
  → final_generic_name = 캐비어

- 엔초비, 엔쵸비, 안초비, 앤초비, anchovy, Anchovy
  → final_generic_name = 엔초비

주의 규칙:

- 엔초비 까나페, 캐비어 까나페처럼 까나페가 본체이면 R98_OUT_OF_SCOPE로 보낸다.
- 캐비어 군함, 캐비어 초밥, 캐비어 스시, 캐비어 롤처럼 초밥/군함/롤이 본체이면 R98_OUT_OF_SCOPE로 보낸다.
- 타코와사비 군함, 타코와사비 초밥처럼 군함/초밥 형태가 본체이면 R98_OUT_OF_SCOPE로 보낸다.
- 보타르가 파스타처럼 파스타/면류가 본체이면 R98_OUT_OF_SCOPE로 보낸다.
- 단품 반찬/안주명으로서의 타코와사비, 고노와다, 보타르가, 캐비어, 엔초비는 R04_해외특수염장류로 처리한다.
- 다른 음식명과 결합되어 본체가 불명확하면 R97_젓갈장아찌류복합검토로 보낸다.

attribute 규칙:
- 토마토, 바질, 레몬생크림, 문어, 낙지, 시로메, 시로미, 사르데냐, 어란, 철갑상어, 캐비아리 등은 attribute에 남긴다.


## R05_장아찌/절임류

family_group = 장아찌/절임류

final_generic_name 후보:
- 장아찌
- 절임
- 피클

INTERNAL_PRIORITY:
1. 피클
2. 장아찌
3. 절임

bucket 판단:
- 장아찌/절임류는 채소, 과일, 해산물 등을 절인 메뉴를 처리한다.
- 피클은 장아찌/절임으로 합치지 않고 final_generic_name = 피클로 보존한다.

## R97_젓갈장아찌류복합검토

family_group = 젓갈장아찌류복합검토

목적:
젓갈/장아찌류 안에는 남을 수 있지만, 기존 R01~R06의 family_group 또는 final_generic_name으로 자동 통합하면 의미 손실이나 category 혼동이 생길 수 있는 후보를 검토용으로 분리한다.

INTERNAL_PRIORITY:
1. 게장/새우장/젓갈/절임/해외특수염장류가 섞인 복합 후보
2. 해외특수염장류 키워드가 들어가지만 다른 음식 형태와 결합되어 본체가 불명확한 후보
3. 젓갈/장아찌류 안에 남을 가능성은 있으나 세부 block 확정이 어려운 후보
4. 젓갈/장아찌류 후보로 볼 여지는 있으나 메뉴 본체 확정이 어려운 후보
5. final_generic_name 후보로 자동 통합하면 의미 손실이 큰 후보

매칭 조건:
- 연어장, 쥬쿠세이숙성연어장, 전복장, 생참치장, 꼬막장, 모둠장, 정우모듬해물장, 꽃게새우간장처럼 게장/새우장/해외특수염장류로 단순 통합하기 어려운 해산물장 후보
- 감태, 바다포도, 날치알새우, 청어알, 트러플청어알처럼 젓갈/염장류 가능성은 있으나 자동 통합이 애매한 후보
- 젓갈/장아찌류 안에 남을 가능성은 있지만 자동 통합하면 의미 손실이 큰 후보

제외 조건:
- 게장류 신호가 명확하면 R01
- 새우장류 신호가 명확하면 R02
- 식해류 신호가 명확하면 R03
- 해외특수염장류 신호가 명확하면 R04
- 장아찌/절임류 신호가 명확하면 R05
- 젓갈류 신호가 명확하면 R06
- 젓갈/장아찌류가 아님이 명확하면 R98
- 젓갈/장아찌류 여부 자체가 불명확하면 R99

출력 규칙:
- assigned_block = R97_젓갈장아찌류복합검토
- final_generic_name = raw_generic_name
- family_group = 젓갈장아찌류복합검토
- normalization_actions = ["broad_merge_review"]
- review_status = needs_review
