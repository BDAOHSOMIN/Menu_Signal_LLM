## 공통 처리 원칙

- BLOCK_PRIORITY를 위에서부터 적용한다.
- 특정 R block에 들어간 뒤에는 해당 block의 INTERNAL_PRIORITY를 위에서부터 적용한다.
- raw_generic_name은 2차 프롬프트에서 생성된 후보명이고, final_generic_name은 3차에서 확정하는 분석용 이름이다.
- final_generic_name 후보 목록은 “강제 통합 가능한 대표명 목록”이다.
- AI가 새로운 assigned_block, family_group, final_generic_name을 임의 생성하면 안 된다.
- final_generic_name = 볶음, final_generic_name = 복합볶음은 사용하지 않는다.
- category rule에 명시되지 않은 음식명을 과도하게 추론하지 않는다.
- 데이터에 실제 등장한 표현, 메뉴명, 조리명, 재료명 신호를 기준으로 판단한다.
- 데이터 신호가 약하거나 음식 정체성이 불명확하면 임의로 통합하지 말고 R97_볶음류복합검토 또는 R99_REVIEW로 보낸다.


## 본체 메뉴명 판단 원칙

- 여러 재료가 함께 등장할 때는 단순히 앞쪽 단어만 보지 않는다.
- 메뉴명의 끝부분 또는 핵심 음식명에 가까운 표현을 우선 확인한다.
- 단, 무조건 뒤 단어만 따르지 않고, 아래 named anchor가 있는지 먼저 판단한다.
- named anchor란 제육볶음, 두루치기, 주물럭, 닭갈비, 닭발, 닭똥집볶음, 곱창볶음, 막창볶음, 대창볶음, 순대볶음, 오돌뼈볶음, 껍데기볶음, 낙지볶음, 쭈꾸미볶음, 오징어볶음, 새우볶음, 잡채처럼 final_generic_name 후보로 강제 통합 가능한 대표 메뉴명을 말한다.
- named anchor가 명확하면 부재료보다 named anchor를 우선한다.
- 부재료는 메뉴 본체를 꾸미는 재료이므로, 부재료만 보고 block을 바꾸지 않는다.

### 부재료보다 본체 anchor를 우선하는 예

- 굴제육볶음, 오징어제육, 오징어제육볶음, 제육낙지볶음처럼 어패류가 붙어도 제육볶음 본체가 명확하면 R06_육류볶음 / final_generic_name = 제육볶음
- 두부제육볶음, 두부제육두루치기, 두부제육김치, 스팸제육두부김치처럼 두부가 붙어도 제육볶음 혹은 두루치기 본체가 명확하면 R06_육류볶음
- 소고기건두부볶음, 우삼겹건두부볶음처럼 건두부가 붙어도 소고기/우삼겹이 본체이면 R06_육류볶음 / final_generic_name = 육류볶음
- 계란고기볶음은 계란이 부재료이고 고기볶음이 본체이면 R06_육류볶음 / final_generic_name = 육류볶음
- 계란 양고기볶음은 계란이 부재료이고 양고기볶음이 본체이면 R06_육류볶음 / final_generic_name = 양고기볶음
- 두부낙지볶음은 두부가 부재료이고 낙지볶음이 본체이면 R07_어패류볶음 / final_generic_name = 낙지볶음
- 두부오징어볶음은 두부가 부재료이고 오징어볶음이 본체이면 R07_어패류볶음 / final_generic_name = 오징어볶음
- 계란새우볶음, 계란새우야채볶음은 계란/야채가 부재료이고 새우볶음이 본체이면 R07_어패류볶음 / final_generic_name = 새우볶음
- 새우당면볶음,해물당면볶음 같이 판단이 어려운 경우는 R97로 보낸다.

### R97로 보내야 하는 복합 후보

- 육류와 어패류가 모두 본체처럼 결합되어 하나의 named anchor로 확정하기 어려우면 R97_볶음류복합검토로 보낸다.
- 낙곱새, 쭈곱새, 오삼볶음, 쭈삼볶음, 낙삼볶음, 불낙볶음은 한쪽 재료가 단순 부재료가 아니라 복합 메뉴명이므로 R97로 보낸다.
- 차돌쭈꾸미, 대창쭈꾸미, 쭈꾸미삼겹, 갑오징어삼겹볶음처럼 육류와 어패류가 함께 본체로 보이면 R97로 보낸다.
- 단, 굴제육볶음, 오징어제육, 두부제육볶음처럼 제육/두루치기/닭갈비/닭발/곱창볶음 등 named anchor가 명확한 경우에는 R97로 보내지 않는다. 최대한 판단하도록 한다.


## block별 fallback 원칙

- R02_중식볶음, R03_일식볶음, R04_동남아식볶음, R05_양식볶음은 해외식/외국식 볶음 후보를 처리한다.
- 단, 해외식 block fallback은 아래 block별 신호가 명확한 경우에만 사용한다.
- 단순히 외국어처럼 보이거나 소스명이 들어갔다고 해외식 fallback으로 보내지 않는다.
- 특히 양식볶음은 매우 보수적으로 판단한다. 실제 대중적인 양식/외국식 메뉴명 또는 영어 메뉴명 신호가 명확한 경우에만 양식볶음 fallback을 사용한다.
- R06_육류볶음, R07_어패류볶음, R08_채소볶음, R09_곡류두류볶음은 흡수형 fallback을 가진다.
- R08_채소볶음과 R09_곡류두류볶음은 R06_육류볶음, R07_어패류볶음보다 뒤에서 판단한다.
- 채소/곡류/두류 재료가 포함되어도 육류/어패류가 본체이면 채소볶음이나 곡류두류볶음으로 보내지 않는다.
- 예: 콩나물소고기볶음, 마늘쫑차돌볶음, 숙주삼겹볶음 → 콩나물/마늘쫑/숙주는 부재료이고 소고기/차돌/삼겹이 본체이면 R06_육류볶음
- 예: 전복버터숙주볶음, 문어김치볶음, 새우가지볶음 → 숙주/김치/가지가 부재료이고 전복/문어/새우가 본체이면 R07_어패류볶음

## 다른 category로 보내야 하는 후보

- 볶음밥, 김치볶음밥, 새우볶음밥, 계란볶음밥, 나시고랭, 카오팟, 카우팟, 까오팟은 밥류 본체이므로 R98_OUT_OF_SCOPE로 보낸다.
- 볶음면, 볶음우동, 볶음쌀국수, 쌀국수볶음, 야끼소바, 야끼우동, 팟타이, 팟씨유는 면류 본체이므로 R98_OUT_OF_SCOPE로 보낸다.
- 닭볶음탕, 닭도리탕, 곱도리탕은 국/탕류 또는 찜류 성격이 강하므로 R98_OUT_OF_SCOPE로 보낸다.
- 불고기, 소불고기, 돼지불고기, 오리불고기, 한우불고기, 고추장불고기, 닭불고기, 돼지불백은 구이류로 보내기로 했으므로 R98_OUT_OF_SCOPE로 보낸다.
- 양장피는 나물,무침류/냉채류 후보이므로 R98_OUT_OF_SCOPE로 보낸다.
- 칠리새우, 크림새우, 깐풍새우, 깐풍기, 유린기, 탕수육처럼 튀김 본체에 소스가 결합된 후보는 R98_OUT_OF_SCOPE 또는 R97_볶음류복합검토로 보낸다.
- 찌개, 전골, 조림, 찜, 구이, 튀김, 전, 부침, 무침, 회, 숙회가 본체인 후보는 R98_OUT_OF_SCOPE로 보낸다.
- 닭발구이, 숯불닭발, 직화닭발, 닭갈비구이, 숯불닭갈비, 직화닭갈비, 닭똥집구이, 닭근위구이, 모래집구이처럼 구이/숯불/직화/구운 신호가 명확한 후보는 R98_OUT_OF_SCOPE로 보낸다.
- 단, 구이/숯불/직화/구운 신호가 없는 닭발, 닭갈비, 닭똥집, 닭근위, 모래집 단품은 볶음류 안에서 처리한다.
- 구이 신호가 없는 닭갈비,닭똥집,닭발은 기본값을 볶음류로 판단한다.
- 즉, 닭발 단독 → R06_육류볶음 / final_generic_name = 닭발
- 닭갈비 단독 → R06_육류볶음 / final_generic_name = 닭갈비
- 닭똥집 단독 → R06_육류볶음 / final_generic_name = 닭똥집볶음


## R98_OUT_OF_SCOPE

family_group = ""

목적:
볶음류가 아니거나 다른 category가 더 적절한 후보를 분리한다.

매칭 조건:
- 볶음밥, 나시고랭, 카오팟, 카우팟, 까오팟 등 밥류 본체
- 볶음면, 볶음우동, 볶음쌀국수, 쌀국수볶음, 야끼소바, 야끼우동, 팟타이, 팟씨유 등 면류 본체
- 닭볶음탕, 닭도리탕, 곱도리탕 등 탕/찜류 본체
- 불고기/불백 계열
- 양장피
- 튀김+소스 본체가 강한 칠리새우, 크림새우, 깐풍기, 유린기, 탕수육 등
- 구이/찜/조림/전골/찌개/전/부침/무침/회/숙회 등 다른 조리법 본체

출력 규칙:
- assigned_block = R98_OUT_OF_SCOPE
- final_generic_name = raw_generic_name
- family_group = ""
- normalization_actions = ["exclude_or_correction"]
- review_status = correction_candidate


## R01_떡볶이류

family_group = 떡볶이류

final_generic_name 후보:
- 떡볶이

INTERNAL_PRIORITY:
1. 떡볶이

bucket 판단:
- 떡볶이류는 떡볶이, 라볶이, 쫄볶이 등 떡볶이 계열 메뉴를 처리한다.
- 떡볶이류는 육류볶음/어패류볶음보다 먼저 처리한다.
- 해물떡볶이, 곱창떡볶이, 차돌떡볶이처럼 육류/어패류 재료가 붙어도 본체가 떡볶이면 떡볶이류로 처리한다.

통합 규칙:
- 떡볶이, 국물떡볶이, 즉석떡볶이, 로제떡볶이, 크림떡볶이, 마라떡볶이, 짜장떡볶이, 해물떡볶이, 곱창떡볶이, 차돌떡볶이, 라볶이, 쫄볶이
  → final_generic_name = 떡볶이

주의 규칙:
- 떡꼬치처럼 떡볶이가 아닌 메뉴는 R98_OUT_OF_SCOPE 또는 R97_볶음류복합검토로 보낸다.

bucket 판단:

- 대표성이 높은 마라샹궈, 팔보채, 마파두부, 유산슬만 별도 final_generic_name으로 살린다.
- 나머지 중식 고유 볶음/소스요리 후보는 final_generic_name = 중식볶음으로 통합한다.
- 아래 신호는 “해당 block으로 자동 통합할 신호”와 “R98/R97로 제외해야 할 신호”를 구분한다.
- 데이터에 등장했다는 이유만으로 모두 볶음류로 처리하지 않는다.

통합 규칙:
- 마라샹궈, 마라상궈, 마라샹구어, 마라상구어, 로제마라샹궈

- 유산슬, 류산슬, 유산슬볶음, 류산슬볶음, 게살유산슬

- 마라룽샤, 마라롱샤, 궁보/궁바오/쿵파오 계열, 어향 계열, 지삼선, 경장육슬/경장육사 계열, 라즈지/라자지 계열, 라조육 계열, 난자완스, 후이궈러우/회과육, 깐궈/간궈 계열, 향라/샹라 계열, 홍소 계열, 흑후추 계열, 사천식, 중화풍, 광동식 해물볶음, 토마토계란볶음

fallback 신호:

주의 규칙:

- 중식 신호가 약하거나 단순히 소스명만 있는 경우에는 중식볶음으로 과도하게 추론하지 말고 R97로 보낸다.

## R97_볶음류복합검토

family_group = 볶음류복합검토

목적:
볶음류 안에는 남을 수 있지만, 기존 R01~R09의 family_group 또는 final_generic_name 후보로 자동 통합하면 의미 손실이나 category 혼동이 생길 수 있는 후보를 검토용으로 분리한다.

INTERNAL_PRIORITY:
1. 육류 + 어패류가 모두 본체인 복합 후보
2. 부재료가 아니라 여러 볶음류 축이 결합된 후보
3. 해외식 볶음 여부가 애매한 후보
4. 육류/어패류/채소/곡류두류 fallback 중 어느 쪽이 본체인지 불명확한 후보
5. 볶음밥/볶음면/탕/전골/구이 등 다른 category와 경계가 애매한 후보
6. final_generic_name 후보로 자동 통합하면 의미 손실이 큰 후보

매칭 조건:
- 낙곱새, 쭈곱새처럼 육류/부속과 어패류가 함께 본체인 후보
- 오삼볶음, 쭈삼볶음, 낙삼볶음, 불낙볶음처럼 육류와 어패류가 함께 본체인 후보
- 차돌쭈꾸미, 대창쭈꾸미, 갑오징어삼겹볶음처럼 육류와 어패류가 결합되어 단일 named anchor로 확정하기 어려운 후보
- 해외식 볶음처럼 보이지만 block별 fallback 신호로 확정하기 어려운 후보
- 양식/외국식 여부가 애매하고 단순 소스명만 있는 후보
- 채소와 육류/어패류가 섞여 주재료 판단이 어려운 후보
- 철판볶음처럼 볶음류는 맞지만 재료축이 불명확한 후보
- 볶음류 안에 남을 가능성은 있지만 자동 통합하면 의미 손실이 큰 후보

제외 조건:
- 제육/두루치기/닭갈비/닭발/닭똥집/곱창볶음/막창볶음/대창볶음/순대볶음 등 육류 named anchor가 명확하면 R06
- 낙지볶음/쭈꾸미볶음/오징어볶음/새우볶음 등 어패류 named anchor가 명확하면 R07
- 떡볶이류 신호가 명확하면 R01
- 중식볶음 신호가 명확하면 R02
- 일식볶음 신호가 명확하면 R03
- 동남아식볶음 신호가 명확하면 R04
- 양식볶음 신호가 명확하면 R05
- 채소볶음 신호가 명확하면 R08
- 곡류두류볶음 신호가 명확하면 R09
- 볶음류가 아님이 명확하면 R98
- 볶음류 여부 자체가 불명확하면 R99

출력 규칙:
- assigned_block = R97_볶음류복합검토
- final_generic_name = raw_generic_name
- family_group = 볶음류복합검토
- normalization_actions = ["broad_merge_review"]
- review_status = needs_review
