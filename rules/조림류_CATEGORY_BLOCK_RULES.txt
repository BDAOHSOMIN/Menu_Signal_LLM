# CATEGORY_BLOCK_RULES: 조림류

## BLOCK_PRIORITY

R98_OUT_OF_SCOPE  
R01_어패류조림  
R02_육류조림  
R03_난류조림  
R04_채소조림  
R97_조림류복합검토  
R99_REVIEW  

## 공통 처리 원칙

- BLOCK_PRIORITY를 위에서부터 적용한다.
- 특정 R block에 들어간 뒤에는 해당 block의 INTERNAL_PRIORITY를 위에서부터 적용한다.
- raw_generic_name은 2차 프롬프트에서 생성된 후보명이고, final_generic_name은 3차에서 확정하는 분석용 이름이다.
- final_generic_name 후보 목록은 “강제 통합 가능한 대표명 목록”이다.
- AI가 새로운 assigned_block, family_group, final_generic_name을 임의 생성하면 안 된다.
- final_generic_name = 조림은 사용하지 않는다.

- 조림류에서는 세부 재료명을 final_generic_name으로 과도하게 살리지 않는다.
- 생선류 조림은 final_generic_name = 생선조림으로 통합한다.
- 생선 외 어패류 조림은 final_generic_name = 어패류조림으로 통합한다.
- 장조림 신호가 명확하면 final_generic_name = 장조림으로 통합한다.
- 장조림이 아닌 육류 기반 조림은 final_generic_name = 육류조림으로 통합한다.
- 난류 기반 조림은 final_generic_name = 난류조림으로 통합한다.
- 두부 기반 조림은 final_generic_name = 두부조림으로 통합한다.
- 두부 외 채소 기반 조림은 final_generic_name = 채소조림으로 통합한다.

- 세부 재료명, 지역명, 산지, 양념명, 조리 스타일, 해외 조림 신호, 정식/백반/쌈/수량 표현은 final_generic_name에 넣지 않고 attribute에 남긴다.
- 시래기, 우거지, 무, 묵은지, 김치, 간장, 고추장, 매콤, 매운, 정식, 백반, 쌈, 대/소/인분 같은 표현은 대체로 본체가 아니라 부재료·양념·제공형태로 본다.
- 이런 부재료·양념·제공형태가 붙어도 조림 본체가 명확하면 R97로 보내지 않고 해당 block에서 자동 처리한다.

## 복합 조림 판단 원칙
- 재료가 2개 이상 있다고 해서 무조건 R97로 보내지 않는다.
- 먼저 메뉴명 안에서 조림 본체가 무엇인지 판단한다.
- 조림 본체가 명확하면 해당 block으로 처리하고, 나머지 재료는 attribute에 남긴다.
- 조림 본체가 명확하지 않거나, 두 개 이상의 주재료 축이 비슷하게 강해서 자동 통합하면 의미 손실이 클 경우 R97_조림류복합검토로 보낸다.

- 예를 들어 시래기코다리조림, 묵은지고등어조림, 고등어무조림처럼 생선조림 본체가 명확하면 생선조림으로 처리한다.
- 갑오징어코다리조림, 전복코다리조림, 문어코다리조림처럼 생선조림 본체가 비교적 명확하면 생선조림으로 처리하고, 갑오징어/전복/문어는 attribute에 남긴다.
- 표고달걀조림처럼 난류조림 본체가 명확하면 난류조림으로 처리하고, 표고는 attribute에 남긴다.
- 소고기감자조림처럼 감자조림 본체로 읽히거나 육류/채소 중 어느 축이 본체인지 애매한 후보는 R97로 보낼 수 있다.
- 닭고기버섯조림처럼 버섯조림인지 닭고기조림인지 애매한 후보는 R97로 보낼 수 있다.
- 차돌두부조림처럼 두부조림 본체로도 보이지만 차돌 신호가 강해 자동 통합 시 의미 손실이 우려되는 후보는 R97로 보낸다.
- 갈비명태조림, 소갈비명태조림, 규스지전복조림처럼 육류와 어패류가 강하게 결합해 주재료 축이 충돌하는 후보는 R97로 보낸다.

- R97은 실패 처리용이 아니라, 자동 통합하면 의미 손실이 생길 수 있는 후보를 사람이 검수하기 위한 block이다.
- R97로 보낸 후보는 final_generic_name = raw_generic_name으로 유지한다.

## R98_OUT_OF_SCOPE

family_group = ""

목적:  
조림류가 아니거나 다른 category가 더 적절한 후보를 분리한다.

매칭 조건:
- 간장게장, 양념게장, 꽃게장, 새우장, 간장새우, 간장대하, 꼬막장, 간장꼬막, 우렁장처럼 장류/절임류 본체
- 계란장, 달걀장, 반숙계란장, 마약계란장, 간장계란장처럼 난류 간장절임 본체
- 초절임고등어, 고등어초절임처럼 초절임/식초절임 계열 본체
- 보리굴비, 고추장보리굴비, 남도보리굴비, 고등어자반, 자반고등어, 반건조 포류처럼 조림류가 아닌 후보
- 강된장, 해물강된장처럼 장/소스류 또는 찌개류 본체
- 라구, 라구볼로네제, 푸타네스카처럼 소스/파스타 계열 본체
- 가지덴가쿠, 된장가지처럼 조림보다 구이/된장구이 성격이 강한 후보
- 구이, 찜, 볶음, 탕, 국, 찌개, 전골, 튀김, 전, 부침, 무침, 회처럼 조림류가 아닌 조리법 본체
- 소스명, 토핑명, 추가, 옵션, 비메뉴 설명문

처리 기준:
- 조림 본체가 명확한 정식/백반/쌈 메뉴는 R98로 보내지 않는다.
- 예: 갈치조림정식, 코다리조림백반, 두부조림정식은 해당 조림 block으로 보내고 정식/백반은 attribute에 남긴다.
- 게장정식, 꼬막정식, 보리굴비정식처럼 본체가 조림류가 아니면 R98로 보낸다.

- final_generic_name = raw_generic_name
- family_group = ""
- normalization_actions = ["exclude_or_correction"]
- review_status = correction_candidate

## R01_어패류조림

family_group = 어패류조림

final_generic_name 후보:
- 생선조림
- 어패류조림

INTERNAL_PRIORITY:
1. 생선조림
2. 어패류조림

bucket 판단:
- R01_어패류조림은 생선, 갑각류, 패류, 연체류 등 어패류를 간장이나 양념에 조린 후보를 처리한다.
- 생선류 조림 본체가 명확하면 final_generic_name = 생선조림으로 통합한다.
- 생선 외 어패류 조림 본체가 명확하면 final_generic_name = 어패류조림으로 통합한다.
- 세부 생선명과 어패류명은 attribute에 남긴다.
- 지역명, 산지, 크기, 수량, 정식/백반/쌈, 양념 스타일은 attribute에 남긴다.
- 홍소/훙소/훙쏘/미소니츠케/쇼유니츠케 같은 해외 조림 신호가 있어도 생선/어패류 축이 명확하면 R01로 처리하고 해당 신호는 attribute에 남긴다.
- 생선조림 본체가 명확하면 앞에 붙은 시래기, 묵은지, 무, 김치, 갑오징어, 문어, 전복, 해물 등은 attribute로 남긴다.
- 단, 육류와 어패류가 강하게 결합해 어느 축이 본체인지 애매하면 R97로 보낸다.

통합 규칙:
- 갈치조림, 고등어조림, 코다리조림, 명태조림, 병어조림, 도루묵조림, 가오리조림, 가자미조림, 삼치조림, 대구조림, 도미조림, 우럭조림, 장치조림, 장어조림, 멜조림, 조기조림, 민어조림, 볼락조림, 생선조림
  → final_generic_name = 생선조림

- 시래기코다리조림, 코다리시래기조림, 묵은지고등어조림, 고등어무조림, 시래기명태조림, 갈치시래기조림처럼 부재료가 붙었지만 생선조림 본체가 명확한 후보
  → final_generic_name = 생선조림

- 갑오징어코다리조림, 전복코다리조림, 문어코다리조림, 해물코다리조림처럼 기타 어패류가 붙었지만 생선조림 본체가 비교적 명확한 후보
  → final_generic_name = 생선조림

- 홍소갈치, 훙쏘명태, 훙쏘우조기, 사바노미소니츠케처럼 해외 조림 신호가 있지만 생선축이 명확한 후보
  → final_generic_name = 생선조림

- 갑오징어조림, 오징어조림, 문어조림, 전복조림, 조개조림, 골뱅이조림, 새우조림, 게조림, 꽃게조림처럼 생선 anchor 없이 생선 외 어패류가 본체인 후보
  → final_generic_name = 어패류조림

R97로 보내는 경우:
- 갈비명태조림, 소갈비명태조림, 규스지전복조림처럼 육류와 어패류가 강하게 결합해 주재료 축이 육류조림인지 어패류조림인지 충돌하는 후보
- 생선명 단독으로 조림 신호가 약한 후보
- 모듬조림처럼 재료축이 불명확한 후보

attribute 규칙:
- 생선명, 어패류명, 시래기, 우거지, 무, 묵은지, 김치, 간장, 매콤, 홍소, 훙쏘, 미소니츠케, 쇼유니츠케, 정식, 백반, 쌈 등 세부 재료와 스타일은 attribute에 남긴다.

## R02_육류조림

family_group = 육류조림

final_generic_name 후보:
- 장조림
- 육류조림

INTERNAL_PRIORITY:
1. 장조림
2. 육류조림

bucket 판단:
- R02_육류조림은 소고기, 돼지고기, 닭고기, 부속, 스지, 꼬리, 갈비 등 육류를 간장이나 양념에 조린 후보를 처리한다.
- 장조림은 final_generic_name = 장조림으로 통합한다.
- 장조림이 아닌 육류 기반 조림은 final_generic_name = 육류조림으로 통합한다.
- 세부 육류 재료명, 부위명, 조리 스타일, 해외 조림 신호는 attribute에 남긴다.
- 홍사오육, 홍사러우, 홍소육, 쇼유니츠케, 미소조림 등 해외 조림 신호가 있어도 육류 축이 명확하면 R02로 처리한다.
- 채소나 버섯이 부재료처럼 붙어도 육류조림 본체가 명확하면 R02로 처리한다.
- 반대로 육류가 들어갔지만 채소조림인지 육류조림인지 애매하면 R97로 보낸다.
- 육류와 어패류가 강하게 결합한 후보는 R97로 보낸다.

통합 규칙:
- 장조림, 소고기장조림, 한우장조림, 돼지고기장조림, 사태장조림, 소고기메추리알장조림
  → final_generic_name = 장조림

- 간장닭조림, 닭발조림, 닭날개윙조림, 삼겹간장조림, 항정간장조림, 소갈비살조림, 소고기조림, 소꼬리조림, 스지조림, 곱창조림, 돼지곱창조림, 소세지간장조림, 홍사오육, 홍사러우
  → final_generic_name = 육류조림

- 닭고기버섯조림, 소고기감자조림처럼 육류와 채소가 함께 있으나 어느 쪽이 본체인지 애매하면 R97로 보낼 수 있다.
- 차돌두부조림처럼 두부조림 본체로도 보이고 차돌 신호도 강한 후보는 R97로 보낸다.

R97로 보내는 경우:
- 육류 + 어패류가 강하게 결합한 후보
- 육류 + 두부/채소가 결합했는데 본체 판단이 애매한 후보
- 자동으로 육류조림에 넣으면 의미 손실이 클 수 있는 후보

attribute 규칙:
- 소고기, 한우, 돼지고기, 닭, 삼겹, 항정, 갈비, 소꼬리, 스지, 곱창, 소세지, 감자, 버섯, 두부, 홍사오, 쇼유니츠케 등 세부 재료와 스타일은 attribute에 남긴다.

## R03_난류조림

family_group = 난류조림

final_generic_name 후보:
- 난류조림

INTERNAL_PRIORITY:
1. 난류조림

bucket 판단:
- R03_난류조림은 달걀, 계란, 메추리알 등을 간장이나 양념에 조린 후보를 처리한다.
- 계란조림, 달걀조림, 메추리알조림, 메추리알장조림은 final_generic_name = 난류조림으로 통합한다.
- 표고달걀조림처럼 채소/버섯이 함께 있어도 난류조림 본체가 명확하면 R03으로 처리한다.
- 표고, 버섯 등 부재료는 attribute에 남긴다.
- 소고기메추리알장조림처럼 육류 장조림 신호가 강하면 R02_육류조림의 장조림으로 처리한다.
- 계란장, 달걀장, 반숙계란장, 마약계란장, 간장계란장은 조림류가 아니므로 R98로 보낸다.
- 계란찜, 계란후라이, 계란말이는 조림류가 아니므로 R98로 보낸다.

통합 규칙:
- 계란조림, 달걀조림, 메추리알조림, 메추리알장조림, 표고달걀조림
  → final_generic_name = 난류조림

attribute 규칙:
- 계란, 달걀, 메추리알, 표고, 버섯 등 세부 재료는 attribute에 남긴다.

## R04_채소조림

family_group = 채소조림

final_generic_name 후보:
- 두부조림
- 채소조림

INTERNAL_PRIORITY:
1. 두부조림
2. 채소조림

bucket 판단:
- R04_채소조림은 두부, 감자, 연근, 가지, 표고, 버섯, 무, 시래기, 우거지 등 두부·채소 기반 조림 후보를 처리한다.
- 앞쪽 R01_어패류조림, R02_육류조림, R03_난류조림에서 명확하게 처리되는 후보는 R04로 보내지 않는다.
- 두부가 단독 본체이거나 양념·지역명·제공형태 정도만 결합된 후보는 final_generic_name = 두부조림으로 통합한다.
- 두부 외 채소가 단독 본체이거나 양념·지역명·제공형태 정도만 결합된 후보는 final_generic_name = 채소조림으로 통합한다.
- 두부조림은 일반 채소조림보다 우선한다.
- 세부 채소명, 양념, 조리 스타일, 제공형태는 attribute에 남긴다.
- 홍소가지, 훙소가지처럼 해외 조림 신호가 있어도 가지/채소 축이 명확하면 채소조림으로 처리하고, 홍소/훙소 신호는 attribute에 남긴다.
- 두부/채소에 육류·어패류가 강하게 결합해 본체 판단이 애매하면 R97로 보낸다.

통합 규칙:
- 두부조림, 간장두부, 들기름두부조림, 대전식매운두부, 두부조림정식
  → final_generic_name = 두부조림

- 가지조림, 감자조림, 연근조림, 표고조림, 버섯조림, 무조림, 시래기조림, 우거지조림, 고추조림, 배추조림, 베이크드빈스, 홍소가지, 훙소가지
  → final_generic_name = 채소조림

R97로 보내는 경우:
- 차돌두부조림처럼 두부조림 본체로도 보이지만 다른 주재료 신호가 강해 자동 통합이 애매한 후보
- 소고기감자조림, 닭고기버섯조림처럼 채소조림인지 육류조림인지 애매한 후보
- 두부/채소와 다른 주재료가 결합해 의미 손실이 우려되는 후보

attribute 규칙:
- 두부, 간장, 들기름, 대전식, 매운, 정식, 감자, 연근, 가지, 표고, 버섯, 무, 시래기, 우거지, 고추, 배추, 베이크드빈스, 홍소, 훙소 등 세부 재료와 스타일은 attribute에 남긴다.

## R97_조림류복합검토

family_group = 조림류복합검토

목적:  
조림류 안에는 남을 수 있지만, 기존 R01~R04의 final_generic_name 후보로 자동 통합하면 의미 손실이나 category 혼동이 생길 수 있는 후보를 검토용으로 분리한다.

매칭 조건:
- 주재료 축이 2개 이상 강하게 결합해 어느 block으로 보낼지 애매한 후보
- 육류 + 어패류가 결합한 후보
- 육류 + 두부/채소가 결합했는데 본체 판단이 애매한 후보
- 어패류 + 두부/채소가 결합했는데 본체 판단이 애매한 후보
- 모듬조림, 왕특대조림, 조림처럼 재료축이 불명확한 후보
- 갈치, 고등어, 코다리처럼 단독 재료명만 있고 조림 신호가 약한 후보
- 해외 음식명인데 재료축 또는 조림류 여부가 불명확한 후보
- 자동 통합하면 의미 손실이 클 수 있는 후보

R97로 보내지 않는 후보:
- 시래기, 묵은지, 무, 김치, 우거지, 간장, 매콤, 정식, 백반, 쌈처럼 부재료·양념·제공형태만 붙은 조림 후보
- 생선조림 본체가 명확한 후보
- 육류조림 본체가 명확한 후보
- 난류조림 본체가 명확한 후보
- 두부조림/채소조림 본체가 명확한 후보
- 조림류가 아님이 명확해 R98로 보내야 하는 후보

출력 규칙:
- assigned_block = R97_조림류복합검토
- final_generic_name = raw_generic_name
- family_group = 조림류복합검토
- normalization_actions = ["broad_merge_review"]
- review_status = needs_review

## R99_REVIEW

family_group = ""

목적:  
조림류 여부, 음식명 여부, 또는 적용 가능한 block 자체가 불명확한 후보를 최종 검토한다.

매칭 조건:
- 어떤 block에도 안정적으로 들어가지 않는 경우
- raw_generic_name만으로 조림류인지 알 수 없는 경우
- 음식명인지 옵션/소스/재료명인지 불명확한 경우
- 오타가 심해 조림류 여부 판단이 어려운 후보
- 브랜드명 또는 업장 고유명처럼 보이나 조림류 본체가 드러나지 않는 후보
- 재료가 불명확하거나 어느 조림류 축에도 확실히 속하지 않는 후보

출력 규칙:
- assigned_block = R99_REVIEW
- final_generic_name = raw_generic_name
- family_group = ""
- normalization_actions = ["broad_merge_review"]
- review_status = needs_review
