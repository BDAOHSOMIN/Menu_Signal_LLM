# CATEGORY_BLOCK_RULES: 회류

## BLOCK_PRIORITY

R98_OUT_OF_SCOPE
R01_육류회
R02_어패류회
R97_회류복합검토
R99_REVIEW


## 공통 처리 원칙

- BLOCK_PRIORITY를 위에서부터 적용한다.
- 특정 R block에 들어간 뒤에는 해당 block의 INTERNAL_PRIORITY를 위에서부터 적용한다.
- raw_generic_name은 2차 프롬프트에서 생성된 후보명이고, final_generic_name은 3차에서 확정하는 분석용 이름이다.
- final_generic_name 후보 목록은 “강제 통합 가능한 대표명 목록”이다.
- raw_generic_name이 block 안의 final_generic_name 후보로 명확히 통합 가능하면 반드시 해당 final_generic_name으로 통합한다.
- final_generic_name 후보에 없더라도, raw_generic_name이 특정 R block/family_group에 명확히 속하면 새로운 넓은 대표명으로 억지 통합하지 않는다.
- 이 경우 final_generic_name = raw_generic_name으로 유지하고, family_group은 해당 R block의 family_group으로 출력하며, normalization_actions = ["broad_merge_review"], review_status = needs_review로 둔다.
- R97_회류복합검토와 R99_REVIEW는 검토용 block이므로 final_generic_name = raw_generic_name으로 유지한다.
- AI가 새로운 assigned_block이나 family_group을 임의 생성하면 안 된다.
- AI가 CATEGORY_BLOCK_RULES에 없는 넓은 final_generic_name을 임의 생성하면 안 된다.
- 회류가 아닌 후보는 R98_OUT_OF_SCOPE로 보낸다.
- 회류는 맞지만 육류회/어패류회가 섞여 자동 통합이 위험한 후보는 R97_회류복합검토로 보낸다.
- 회류 여부, 음식명 여부, 적용 가능한 block 자체가 불명확하면 R99_REVIEW로 보낸다.
- 육회, 물회, 막회, 숙회, 세꼬시, 참치, 사시미, 카르파치오, 타르타르, 타다끼 등은 대부분 회류 판단 신호이다.
- 단, 회무침, 회덮밥, 초밥, 구이, 탕, 찜, 조림처럼 다른 조리법/메뉴 본체가 명확하면 회류로 유지하지 않는다.
- 사시미는 raw_generic_name에 사시미/sashimi 신호가 명확한 어패류 회 메뉴에만 적용한다.
- 육류든 어패류든 사시미 신호가 있으면 먼저 사시미로 배정한다.
- 육사시미, 한우사시미, 소고기사시미처럼 육류 신호가 명확한 사시미 후보는 R01_육류회에서 먼저 처리한다.
- 사시미 신호가 없는 참치를 나타내는 참치/마구로/도로/아카미 계열은 final_generic_name = 참치회로 처리한다.
- 사시미 신호가 없고 참치계열을 제외한 일반 생선 단품 회는 final_generic_name = 생선회로 처리한다.
- 사시미 신호가 없는 생선 제외 해산물 단품 회는 final_generic_name = 해산물단품으로 처리한다.
- 일본식 재료명/안주명만 있는 후보는 사시미로 강제 통합하지 않는다.
- 일본식 재료명/안주명만 있고 회류 안에 남을 가능성이 있으면 R02_어패류회에 배정하되 final_generic_name = raw_generic_name으로 유지하고 needs_review로 둔다.
- 고노와다처럼 젓갈/진미류 신호가 강한 후보는 사시미/해산물단품으로 통합하지 않는다. 단독이면 R98_OUT_OF_SCOPE, 회류와 결합된 후보이면 R97_회류복합검토로 보낸다.


## R98_OUT_OF_SCOPE

family_group = ""

목적:
회류가 아니거나 다른 category가 더 적절한 후보를 분리한다.

매칭 조건:
- 마끼, 마키, 데마끼, 데마키, 후토마끼, 후토마키, 테마끼, 테마키, 군함, 군함말이는 회류가 아니라 초밥/롤류 성격이 강하므로 R98_OUT_OF_SCOPE로 보낸다.
- 회무침, 물회무침, 골뱅이무침, 오징어무침처럼 무침류 본체
- 초밥, 스시, 사시미초밥, 회덮밥, 참치덮밥, 육회비빔밥, 육회덮밥
- 생선구이, 장어구이, 조개구이, 전복구이
- 해물탕, 매운탕, 생선찜, 아귀찜, 전골, 조림
- 튀김, 전, 부침, 볶음
- 고노와다 단독, 고노와다회무침처럼 젓갈/무침류 본체가 강한 후보
- 소스명, 토핑명, 추가, 세트, 코스, 비메뉴 설명문

출력 규칙:
- assigned_block = R98_OUT_OF_SCOPE
- final_generic_name = raw_generic_name
- family_group = ""
- normalization_actions = ["exclude_or_correction"]
- review_status = correction_candidate


## R01_육류회

family_group = 육류회

final_generic_name 후보:
- 간천엽
- 간
- 천엽
- 육류타르타르
- 육류카르파치오
- 육류타다끼
- 육사시미
- 뭉티기
- 육회

INTERNAL_PRIORITY:
1. 간천엽
2. 간
3. 천엽
4. 육류타르타르
5. 육류카르파치오
6. 육류타다끼
7. 육사시미
8. 뭉티기
9. 육회

bucket 판단:
- 육류회는 고기/내장/부속을 생식 또는 준생식 형태로 제공하는 회류 bucket이다.
- 육회 계열은 final_generic_name = 육회로 통합한다.
- 모듬육회는 육회로 통합한다.
- 육사시미는 사시미라는 단어가 있어도 어패류 사시미가 아니라 육류회 bucket으로 처리한다.
- 육회타다끼처럼 육류 + 타다끼 신호가 명확하면 육류타다끼로 통합한다.
- 육류 타르타르/카르파치오 신호가 명확하면 각각 육류타르타르, 육류카르파치오로 통합한다.
- 육류회는 명확하지만 위 final_generic_name 후보로 통합하기 어려운 후보는 final_generic_name = raw_generic_name으로 유지하고 needs_review로 둔다.
- 육류회와 어패류회가 결합된 후보는 R97_회류복합검토로 보낸다.

통합 규칙:
- 간천엽, 간 천엽, 간/천엽
  → final_generic_name = 간천엽

- 간, 생간
  → final_generic_name = 간

- 천엽, 생천엽
  → final_generic_name = 천엽

- 육류타르타르, 소고기타르타르, 한우타르타르, 비프타르타르, beef tartare
  → final_generic_name = 육류타르타르

- 육류카르파치오, 소고기카르파치오, 한우카르파치오, 비프카르파치오, beef carpaccio
  → final_generic_name = 육류카르파치오

- 육류타다끼, 소고기타다끼, 한우타다끼, 육회타다끼, 비프타다끼, beef tataki
  → final_generic_name = 육류타다끼

- 육사시미, 소고기사시미, 한우사시미, 생고기사시미, 차돌사시미
  → final_generic_name = 육사시미

- 뭉티기, 뭉티기고기, 생고기뭉티기
  → final_generic_name = 뭉티기

- 육회, 한우육회, 소고기육회, 모듬육회, 육회모듬, 육회한접시
  → final_generic_name = 육회

주의 규칙:
- 육회낙지탕탕이, 육회물회, 한우물회, 육회연어처럼 어패류회 또는 물회와 결합된 후보는 R97_회류복합검토로 보낸다.
- 육회비빔밥, 육회덮밥, 육회초밥은 R98_OUT_OF_SCOPE로 보낸다.
- 사시미라는 단어가 있어도 육류 신호가 명확하면 육사시미로 처리한다.
- 어패류 사시미는 R02_어패류회에서 사시미로 처리한다.
- 어패류 타다끼는 R02_어패류회로 보낸다.

cuisine_signal 규칙:
- 육회, 뭉티기, 육사시미, 간, 천엽, 간천엽은 cuisine_signal = 한식으로 둘 수 있다.
- 육류타다끼는 cuisine_signal = 일식으로 둘 수 있다.
- 육류타르타르, 육류카르파치오는 cuisine_signal = 유럽식으로 둘 수 있다.
- 불명확하면 cuisine_signal = none

attribute 규칙:
- 한우, 소고기, 생고기, 비프, 모듬, 부위명, 조리 스타일은 attribute에 남긴다.


## R02_어패류회

family_group = 어패류회

final_generic_name 후보:
- 물회
- 막회
- 숙회
- 세비체
- 어패류카르파치오
- 어패류타르타르
- 어패류타다끼
- 세꼬시
- 사시미
- 해산물모듬
- 모듬회
- 참치회
- 생선회
- 해산물단품
- 회

INTERNAL_PRIORITY:
1. 물회
2. 막회
3. 숙회
4. 세비체
5. 어패류카르파치오
6. 어패류타르타르
7. 어패류타다끼
8. 세꼬시
9. 사시미
10. 해산물모듬
11. 모듬회
12. 참치회
13. 생선회
14. 해산물단품
15. 회

bucket 판단:
- 세꼬시는 조리 형태에 더 가깝기 때문에 "세꼬시 막회, 물회" 처럼 동시에 등장하면 막회,물회 형태를 더 따르도록 한다.
- 회무침은 회류가 아니라 무침류이므로 R98_OUT_OF_SCOPE로 보낸다.
- 물회 계열은 final_generic_name = 물회로 통합한다. 단, 한우물회/육회물회/생고기물회처럼 육류회 성격이 섞인 후보는 R97_회류복합검토로 보낸다.
- 막회 계열은 final_generic_name = 막회로 통합한다.
- 사시미는 raw_generic_name에 사시미/sashimi 신호가 명확한 어패류 회 메뉴에만 적용한다. (육회 회는 육사시미를 적용할 것)
- 광어사시미, 연어사시미, 도미사시미, 모듬사시미, 참치사시미, 도로사시미처럼 재료명이나 모듬 신호가 있어도 사시미 신호가 명확하면 final_generic_name = 사시미로 통합한다.
- 단, 육사시미/한우사시미/소고기사시미처럼 육류 신호가 명확한 후보는 R01_육류회에서 처리한다.
- 사시미라는 단어가 없는 일본식 재료명/안주명은 사시미로 강제 통합하지 않는다. 엔가와, 안키모, 이쿠라, 우니, 호타테, 아마에비, 시메사바, 마스카와, 유비끼 등과 같은 일식 음식명으로 되어 있는 경우는 은 final_generic_name 후보로 억지 통합하지 않는다. 이 경우 회류 안에 남을 수 있으면 assigned_block = R02_어패류회, family_group = 어패류회, final_generic_name = raw_generic_name, normalization_actions = ["broad_merge_review"], review_status = needs_review로 둔다. 단, 참치/마구로/도로/아카미 같이 참치를 일식명칭으로 표현한 계열은 일본식 재료명 review 규칙에서 예외로 하고, 규정대로 참치회로 통합한다.
- 고노와다는 젓갈/진미류 신호가 강하므로 사시미/해산물단품으로 통합하지 않는다.
- 고노와다 단독 또는 고노와다회무침처럼 젓갈/무침 본체가 강하면 R98_OUT_OF_SCOPE로 보낸다.
- 광어 고노와다, 광어회고노와다, 사시미 고노와다처럼 회와 결합된 고노와다 후보는 R97_회류복합검토로 보낸다.
- 해산물모듬과 모듬회는 명시적으로 모듬 상품 신호가 있을 때만 적용하는 bucket이다.
- raw_generic_name에 모듬/모둠/모듬회/모둠회/플래터/한판/한상 등 명시적 모듬 상품 신호가 있으면 생선회/해산물단품보다 먼저 판단한다. 단, 모듬사시미 같이 사시미 신호가 있으면 사시미가 우선이다.
- 생선회/활어회 계열의 명시적 모듬 상품이면 final_generic_name = 모듬회로 통합한다.
- 즉, 모듬회는 생선명에 모듬키워드가 있을 때 + 해산물모듬은 생선을 제외하고 해산물들에 모듬 신호가 있을때 bucket을 분류한다. 
- 문어, 전복, 멍게, 해삼, 개불, 산낙지, 키조개, 조개류 등 해산물단품 계열의 명시적 모듬 상품이면(문어모듬,낙지모듬 등) final_generic_name = 해산물모듬으로 통합한다.
- 단순 재료명 또는 단품명만으로 모듬을 추론하지 않는다.
- 참치/마구로/도로/아카미 계열은 사시미 신호가 없을 때 final_generic_name = 참치회로 처리한다.
- 광어회, 우럭회, 도미회, 연어회처럼 생선 단품 회 성격이 명확하면 final_generic_name = 생선회로 처리한다.
- 문어회, 전복회, 오징어회, 한치회처럼 생선이 아닌 해산물 단품 회 성격이 명확하면 final_generic_name = 해산물단품으로 처리한다.
- 해산물단품에는 생선을 제외한 메뉴만 넣고, 생선회에는 생선 기반 단품 회만 넣는다.
- 어패류회는 명확하지만 위 final_generic_name 후보로 통합하기 어려운 후보는 final_generic_name = raw_generic_name으로 유지하고 needs_review로 둔다.
- 육류회와 결합된 후보는 R97_회류복합검토로 보낸다.

통합 규칙:
- 물회, 해산물물회, 생선물회, 회물회
  → final_generic_name = 물회

- 막회
  → final_generic_name = 막회

- 숙회, 문어숙회, 오징어숙회, 전복숙회, 해산물숙회
  → final_generic_name = 숙회

- 세비체, ceviche
  → final_generic_name = 세비체

- 어패류카르파치오, 생선카르파치오, 연어카르파치오, 문어카르파치오, 해산물카르파치오, carpaccio
  → final_generic_name = 어패류카르파치오

- 어패류타르타르, 연어타르타르, 참치타르타르, 해산물타르타르, tartare
  → final_generic_name = 어패류타르타르

- 이 경우 회류 안에 남을 수 있으면 assigned_block = R02_어패류회, final_generic_name = raw_generic_name, family_group = 어패류회, normalization_actions = ["broad_merge_review"], review_status = needs_review로 둔다.

주의 규칙:
- 회무침은 R98_OUT_OF_SCOPE로 보낸다.
- 육회물회, 한우물회, 생고기물회는 R97_회류복합검토로 보낸다.
- 초회는 별도 final_generic_name으로 만들지 않고, 재료 본체 기준으로 해산물단품/생선회/해산물모듬에 배정한다.
- 사시미가 들어간 후보는 참치회/생선회/해산물단품/모듬회보다 사시미를 먼저 판단한다.
- 모듬사시미는 모듬회가 아니라 사시미로 처리한다.
- 참치사시미, 도로사시미, 마구로사시미는 참치회가 아니라 사시미로 처리한다.
- 광어사시미, 연어사시미, 도미사시미처럼 재료가 명확한 일반 생선 사시미도 사시미로 처리한다.
- 시메사바, 엔가와, 안키모, 이쿠라, 우니처럼 사시미 신호가 없는 일본식 재료명/안주명은 사시미로 처리하지 않는다.
- 모듬회/해산물모듬은 생선회/해산물단품보다 우선하지만, 반드시 명시적 모듬 상품 신호가 있어야 한다.
- 모듬회가 들어간 후보는 생선회로 보내기 전에 모듬회로 먼저 판단한다.
- 광어모듬회, 우럭모듬회, 광어우럭모듬회처럼 생선명 + 모듬회 구조는 final_generic_name = 모듬회로 통합한다.
- 문어모듬, 전복모듬, 멍게해삼모듬처럼 해산물단품 계열 + 모듬 구조는 final_generic_name = 해산물모듬으로 통합한다.
- 참치/마구로/도로/아카미 계열이라도 사시미 신호가 없을 때만 참치회로 처리한다.
- 참치회 다음에 생선회를 처리하고, 생선회 다음에 해산물단품을 처리한다.
- 생선회에는 생선 기반 단품 회만 들어간다.
- 해산물단품에는 생선을 제외한 해산물 단품 회만 들어간다.
- 육류타르타르/육류카르파치오/육류타다끼는 R01_육류회로 보낸다.

cuisine_signal 규칙:
- 물회, 막회, 숙회, 세꼬시, 생선회, 해산물단품, 모듬회, 해산물모듬은 cuisine_signal = 한식 또는 none으로 둘 수 있다.
- 한식 횟집 맥락이 명확하면 한식, 단순 재료명만 있으면 none으로 둔다.
- 사시미, 참치회는 cuisine_signal = 일식으로 둘 수 있다.
- 어패류타다끼는 cuisine_signal = 일식으로 둘 수 있다.
- 어패류카르파치오, 어패류타르타르는 cuisine_signal = 유럽식으로 둘 수 있다.
- 세비체는 cuisine_signal = 기타해외식으로 둔다.
- 일본식 재료명/안주명 review 후보는 cuisine_signal = 일식으로 둘 수 있다.
- 불명확하면 cuisine_signal = none

attribute 규칙:
- 원재료명, 생선명, 해산물명, 부위명, 숙성, 모듬, 사시미, 도로, 타다끼, 카르파치오, 타르타르, 초회, 일본식 재료명/안주명 등은 attribute에 남긴다.


## R97_회류복합검토

family_group = 회류복합검토

목적:
회류 안에는 남을 수 있지만, 기존 R01~R02의 family_group 또는 final_generic_name 후보로 자동 통합하면 의미 손실이나 category 혼동이 생길 수 있는 후보를 검토용으로 분리한다.

INTERNAL_PRIORITY:
1. 육류회 + 어패류회 복합
2. 물회 + 육류회 복합
3. 탕탕이 + 육회 복합
4. 회류 + 젓갈/진미류 복합
5. 재료 축이 불명확한 복합 회류
6. family_group 자체가 애매한 회류

매칭 조건:
- 육회물회, 한우물회, 생고기물회, 한우육회물회
- 육회낙지탕탕이, 육회낙지탕탕이물회, 산낙육회, 산낙지육회
- 육회바른연어, 육회연어, 육회참치처럼 육류회와 어패류회가 결합된 후보
- 모듬회인데 육류회와 어패류회가 함께 포함된 후보
- 광어 고노와다, 광어회고노와다, 사시미 고노와다, 회고노와다처럼 회류와 고노와다/젓갈성 부재료가 결합된 후보
- raw_generic_name만으로 육류회/어패류회 중 어느 축이 주축인지 불명확한 후보

제외 조건:
- 육류회 신호가 명확하면 R01
- 어패류회 신호가 명확하면 R02
- 회류가 아님이 명확하면 R98
- 회류 여부 자체가 불명확하면 R99

출력 규칙:
- assigned_block = R97_회류복합검토
- final_generic_name = raw_generic_name
- family_group = 회류복합검토
- normalization_actions = ["broad_merge_review"]
- review_status = needs_review


## R99_REVIEW

family_group = ""

목적:
회류 여부, 음식명 여부, 또는 적용 가능한 R block 자체가 불명확한 후보를 최종 검토한다.

매칭 조건:
- 어떤 block에도 안정적으로 들어가지 않는 경우
- raw_generic_name만으로 회류인지 알 수 없는 경우
- 음식명인지 옵션/소스/재료명인지 불명확한 경우
- 오타가 심해 회류 여부 판단이 어려운 후보
- 브랜드명 또는 업장 고유명처럼 보이나 회류 본체가 드러나지 않는 후보
- cuisine_signal은 강하지만 회류 bucket으로 넣기 어려운 후보

출력 규칙:
- assigned_block = R99_REVIEW
- final_generic_name = raw_generic_name
- family_group = ""
- normalization_actions = ["broad_merge_review"]
- review_status = needs_review
