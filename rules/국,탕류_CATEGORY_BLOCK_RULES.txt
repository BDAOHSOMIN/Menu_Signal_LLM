## 공통 처리 원칙

- BLOCK_PRIORITY를 위에서부터 적용한다.
- 특정 R block에 들어가면 해당 block의 INTERNAL_PRIORITY를 위에서부터 적용한다.
- 단, 단어 매칭만으로 final_generic_name을 바로 정하지 않는다.
- 먼저 주재료/음식군 기준으로 assigned_block을 판단하고, 그 다음 해당 block 안에서 INTERNAL_PRIORITY를 적용한다.
- 예: 닭매운탕은 `매운탕` 단어가 있어도 R07_어패류국,탕류가 아니라 R06_가금류국,탕류로 보낸다.
- 복합 메뉴명은 단순 포함 단어가 아니라 핵심 음식명(head food anchor)을 기준으로 판단한다.
- 한국어 복합 메뉴명은 대체로 뒤쪽 음식명이 head 역할을 하므로, 뒤에 오는 음식명에 주의한다.
- 예: 조개어묵탕은 조개탕이 아니라 어묵탕, 꽃게해물탕은 꽃게탕이 아니라 해물탕이다.
- 단, head가 불명확하거나 여러 국,탕류 축이 동등하게 결합된 경우는 R97_국탕류복합검토로 보낸다.
- raw_generic_name은 2차 후보명이고, final_generic_name은 3차 분석용 bucket 이름이다.
- final_generic_name 후보 목록 안에서만 통합한다.
- 새로운 assigned_block, family_group, final_generic_name을 임의 생성하지 않는다.
- final_generic_name 후보에 없지만 block/family_group은 명확하면 final_generic_name = raw_generic_name으로 유지하고 needs_review로 둔다.
- R97_국탕류복합검토와 R99_REVIEW는 검토용 block이므로 final_generic_name = raw_generic_name으로 유지한다.
- 국,탕류가 아니거나 다른 category가 더 적절하면 R98_OUT_OF_SCOPE로 보낸다.
- 국밥류는 국,탕류로 자동 통합하지 않고 R98_OUT_OF_SCOPE로 보낸다.
- R04_일반국,탕류는 named anchor만 처리한다.
- R04 named anchor에 해당하지 않는 일반 국,탕 후보는 `일반국`으로 통합하지 않고 R97_국탕류복합검토로 보낸다.
- R05_육류국,탕류, R06_가금류국,탕류, R07_어패류국,탕류는 block 내부 famil-level fallback을 가진다.
- 육류탕, 가금류탕, 어패류탕은 전체 미매핑 fallback이 아니라 해당 block 내부에서만 사용한다.
- '갈비탕 정식'일 경우 '정식'이어도 '갈비탕'이 명확하므로 '갈비탕'으로 처리할 수 있다.

- 똠양꿍, 똠카가이는 soup로 번역될 수 있어도 국,탕류 valid로 둔다.
- 닭볶음탕, 닭도리탕, 도리탕, 곱도리탕은 '볶음' 단어가 있어도 볶음류가 아니라 R06_가금류국,탕류로 처리한다.
### 출력 규칙

- assigned_block = R98_OUT_OF_SCOPE
- final_generic_name = raw_generic_name
- family_group = ""
- normalization_actions = ["exclude_or_correction"]
- review_status = correction_candidate

- 닭매운탕은 어패류 매운탕이 아니다.
- 닭매운탕은 닭볶음탕이라고 명시된 것이 아니므로 닭볶음탕으로 통합하지 않는다.
- 닭볶음탕/닭도리탕/도리탕이 명시된 경우에만 닭볶음탕으로 처리한다.
- 닭한마리칼국수처럼 면류 본체가 강하면 R98_OUT_OF_SCOPE 또는 R97_국탕류복합검토로 보낸다.

family_group = 어패류국,탕류

### INTERNAL_PRIORITY 해석 원칙
- R07_어패류국,탕류는 생선, 어묵, 해물, 조개, 갑각류, 연체류, 알/곤이, 복어, 추어탕 등 어패류 기반 국/탕 후보를 처리한다.
- 단, 단순히 어패류 재료명이 포함되었다고 모두 R07로 보내지 않는다.
- 메뉴의 본체가 갈비탕, 곰탕, 닭곰탕, 해장국, 면류, 전골류 등 다른 block에 더 적합하면 R07 내부 규칙을 적용하지 않는다.
- INTERNAL_PRIORITY는 단순 키워드 선착순이 아니다.
- 먼저 R07_어패류국,탕류 안에서 처리 가능한 후보인지 판단한다.
- R98_OUT_OF_SCOPE, R01_해외국,탕류, R06_가금류국,탕류 등 다른 block이 더 적절하면 R07 내부 priority를 적용하지 않는다.
- R07 내부 후보가 여러 개 동시에 보일 때 반드시  INTERNAL_PRIORITY를 사용한다.
- 단, priority를 적용하기 전에 각 표현이 아래 중 무엇인지 먼저 판단한다.

### 메뉴명 본체 판단 원칙

R07에서는 단순히 앞 단어/뒤 단어 위치만으로 final_generic_name을 결정하지 않는다.  
먼저 메뉴명 안의 표현을 아래 3가지 기능으로 구분한다.

1. 음식명 head

메뉴의 본체를 결정하는 강한 anchor이다.

예:
- 어묵탕
- 해신탕
- 알탕
- 추어탕
- 연포탕
- 해물탕
- 조개탕

이런 head가 명확하면 앞에 붙은 재료명은 대부분 부재료/강조 재료로 처리한다.

예:
- 조개어묵탕 → 어묵탕
- 꽃게오뎅탕 → 어묵탕
- 해물알탕 → 알탕
- 산낙지해물연포탕 → 연포탕
- 꽃게해물탕 → 해물탕

2. 조리형식 / 국물 스타일

음식명처럼 보이지만, 더 구체적인 음식명 head가 있으면 조리 스타일로만 처리한다.

예:
- 매운탕
- 지리탕
- 맑은탕
- 맑음탕

매운탕/지리탕/맑은탕은 단독이거나 더 구체적인 음식명 head가 없을 때는 final_generic_name이 될 수 있다.

예:
- 대구매운탕 → 매운탕
- 우럭매운탕 → 매운탕
- 대구지리탕 → 지리탕
- 복지리 → 지리탕

하지만 알탕/어묵탕/해물탕/해신탕/연포탕처럼 더 구체적인 음식명 head가 있으면, 매운탕/지리탕은 조리형식으로만 처리한다.

예:
- 알곤이매운탕 → 알탕 즉, 매운탕은 조리형식이므로 알탕보다 약함
- 알고니듬뿍매운탕 → 알탕 
- 동태알고니지리탕 → 알탕

3. 부재료 / 강조 재료

다른 음식명 head 앞에 붙어 본체를 꾸미는 재료명이다.  
부재료는 final_generic_name으로 끌어올리지 않고 attribute로 보존한다.

예:
- 꽃게
- 전복
- 문어
- 낙지
- 조개
- 바지락
- 홍합
- 대구
- 동태
- 해물

예:
- 꽃게해물탕 → final_generic_name = 해물탕, attribute = 꽃게
- 조개어묵탕 → final_generic_name = 어묵탕, attribute = 조개
- 꽃게알탕 → final_generic_name = 알탕, attribute = 꽃게
- 전복해신탕 → final_generic_name = 해신탕, attribute = 전복


### 최종 판단 우선순위

R07 내부에서는 아래 순서로 판단한다.

1. 다른 block으로 보내야 하는지 먼저 확인한다.
2. R07 안에서 처리 가능한 어패류국,탕류인지 확인한다.
3. 메뉴명 안에 강한 음식명 head가 있는지 확인한다.
4. 매운탕/지리탕/맑은탕이 음식명 head인지 조리형식인지 구분한다.
5. 앞쪽 재료명이 본체인지 부재료인지 판단한다.
6. 명확한 음식명 head가 없고 어패류 재료만 남는 경우에만 어패류탕 fallback을 사용한다.

핵심 원칙:

- 음식명 head > 조리형식/국물 스타일 > 부재료/강조 재료 > 어패류탕 fallback
- 끝단어 우선 원칙은 끝단어가 음식명 head일 때 강하게 적용한다.
- 끝단어가 매운탕/지리탕/맑은탕처럼 조리형식이면, 앞쪽에 더 구체적인 음식명 head가 있는지 먼저 확인한다.
- “해물X탕”과 “X해물탕”은 다르게 판단한다.
- “조개탕”은 넓은 패류탕이 아니라, 조개탕 키워드 또는 대중적으로 조개탕으로 인식되는 조개류 head에만 사용한다.

### 주의 규칙

- 복합 후보에서는 조리형식보다 더 구체적인 음식명 head를 우선한다.
- 알곤이지리탕이라면 핵심은 알곤이탕 즉, 알탕이고 지리탕의 형태는 조리형식이기 때문에 알탕을 우선시한다.

- 어패류탕은 R07 안의 family-level fallback이다.
- 전체 미매핑 후보를 어패류탕으로 보내지 않는다.
- 복국, 복탕, 복어국, 복어탕은 생선탕으로 처리한다.

- 복지리, 복지리탕, 복어지리, 참복지리는 생선탕이 아니라 지리탕이다.
- 복매운탕, 복어매운탕은 생선탕이 아니라 매운탕이다.
- 전복탕은 생선탕이 아니라 어패류탕이다. 생선탕은 생선계열만 처리한다.

- 해천탕, 해처탕은 해신탕으로 통합한다.
- 해신탕/해천탕/해처탕 anchor가 명확하면 오리, 갈비, 전복, 문어, 낙지 단어가 있어도 해신탕으로 처리한다.

- 알곤이탕, 알고니탕, 곤이탕, 고니탕은 알탕으로 처리한다.
- 알곤이/알고니/곤이/고니 계열은 매운탕·지리탕·생선탕보다 우선한다.
- 동태알고니지리탕은 지리탕이 아니라 알탕이다.
- 알고니듬뿍매운탕은 매운탕이 아니라 알탕이다.
- 알곤이동태탕은 생선탕이 아니라 알탕이다.

- 추어탕은 추어탕이라는 head가 명확할 때만 처리한다.
- 미꾸라지탕처럼 추어탕 head가 없는 후보는 자동으로 추어탕으로 통합하지 않는다.

- 매운탕은 생선탕으로 보내지 않는다.
- 대구매운탕/우럭매운탕/장어매운탕/복매운탕은 생선탕이 아니라 매운탕이다.
- 닭매운탕, 오리매운탕, 돼지매운탕, 뼈매운탕처럼 비어패류 주재료가 명확하면 R07 매운탕으로 보내지 않는다.

- 지리/맑은탕/맑음탕은 어패류·생선 문맥일 때만 지리탕으로 처리한다.
- 맑은탕/맑음탕은 지리탕으로 통합한다.
- 대구맑은탕은 생선탕이 아니라 지리탕이다.
- 대구지리/우럭지리/아구지리/복지리는 생선탕이 아니라 지리탕이다.
- 맑은곰탕, 맑은닭곰탕처럼 어패류국,탕류가 아닌 후보는 지리탕으로 보내지 않는다.
- 지리와 매운탕이 동시에 들어간 후보는 자동승인하지 않고 R97_국탕류복합검토로 보낸다.

- 연포탕은 연포탕이라는 head가 명확할 때만 처리한다.
- 박속낙지는 연포탕 키워드가 없으면 자동승인하지 않고 R97_국탕류복합검토 또는 needs_review로 둔다.
- 낙지탕, 문어탕, 쭈꾸미탕은 연포탕 키워드가 없으면 연포탕으로 보내지 않는다.

- 해물탕은 raw_generic_name에 해물탕이라는 head가 명확할 때만 처리한다.
- X해물탕은 해물탕으로 처리한다.
- 해물X탕은 X탕 기준으로 판단한다.
- 꽃게해물탕은 해물탕이다.
- 조개해물탕은 조개탕이 아니라 해물탕이다.
- 문어꽃게조개해물탕은 어패류탕이 아니라 해물탕이다.
- 해물꽃게탕은 어패류탕이다.
- 해물대구탕은 생선탕이다.
- 해물알탕은 알탕이다.
- 해물연포탕은 연포탕이다.
- 해물조개탕은 조개탕으로 자동승인하지 않고 어패류탕 또는 needs_review로 둔다.

- 조개어묵탕은 조개탕이 아니라 어묵탕이다.
- 꽃게오뎅탕은 어패류탕이 아니라 어묵탕이다.
- 꽃게알탕은 어패류탕이 아니라 알탕이다.
- 즉, 조개와 꽃게는 어묵탕, 알탕 등에 부재료로 들어간 형태이다.

- 조개탕은 넓은 패류탕이 아니다.
- 조개탕은 조개탕 키워드 또는 대중적으로 조개를 넣은 탕으로 인식되는 조개류 head만 처리한다.
- 바지락탕/홍합탕/백합탕/재첩국/가리비탕은 조개탕으로 처리한다.
- 굴탕, 전복탕, 골뱅이탕, 소라탕은 조개탕으로 처리하지 않고 어패류탕으로 보낸다.
- 낙지바지락탕, 문어조개탕, 산낙지조개탕, 오징어홍합탕, 통골뱅이홍합탕처럼 조개류와 비조개 어패류가 섞인 경우는 조개탕으로 자동승인하지 않고 어패류탕 또는 needs_review로 둔다.

- 대구탕/동태탕/우럭탕/아구탕/장어탕은 별도 final_generic_name으로 살리지 않고 생선탕으로 통합한다.
- 황태해장국/북어해장국은 R02_해장국류가 우선이다.
- 어탕국수처럼 면류 본체가 강하면 R98_OUT_OF_SCOPE 또는 R97_국탕류복합검토로 보낸다.


## R97_국탕류복합검토

family_group = 국탕류복합검토

### INTERNAL_PRIORITY

1. 여러 국,탕류 축이 동등하게 결합된 후보
2. 국밥, 면, 찌개, 전골, 찜 등 다른 category와 경계가 애매한 후보
3. 육류탕, 가금류탕, 어패류탕 fallback으로 자동 통합하면 위험한 후보
4. named anchor 확정이 어려운 일반 국,탕 후보
5. final_generic_name 확정이 어려운 후보

### 매칭 조건

- 국,탕류는 맞지만 R01~R07 중 어디로 보낼지 불명확한 후보
- 콩나물국, 시래기국, 배추국, 우거지국, 감자국처럼 named anchor가 없는 일반 국 후보
- 대구지리매운탕처럼 지리탕과 매운탕 신호가 동시에 있는 후보
- 정식/세트/코스인데 본체 국,탕류가 불명확한 후보

### 출력 규칙

- assigned_block = R97_국탕류복합검토
- final_generic_name = raw_generic_name
- family_group = 국탕류복합검토
- normalization_actions = ["broad_merge_review"]
- review_status = needs_review
