## 공통 처리 원칙

- BLOCK_PRIORITY를 위에서부터 적용한다.
- 특정 R block에 들어가면 해당 block의 INTERNAL_PRIORITY를 위에서부터 적용한다.
- raw_generic_name은 2차 후보명이고, final_generic_name은 3차 분석용 bucket 이름이다.
- final_generic_name 후보 목록 안에서만 통합한다.
- 후보 목록에 없지만 block/family_group은 명확하면 final_generic_name = raw_generic_name으로 유지하고 needs_review로 둔다.
- R97_구이류복합검토와 R99_REVIEW는 검토용 block이므로 final_generic_name = raw_generic_name으로 유지한다.
- 새로운 assigned_block, family_group, final_generic_name을 임의 생성하지 않는다.
- 현재 구이류 밖 후보이면 R98_OUT_OF_SCOPE로 보낸다.


## 조리형 block 우선 원칙

- 스테이크 신호가 명확하면 돼지고기구이/소고기구이/어패류구이보다 R01_스테이크를 우선한다.
- 꼬치 신호가 명확하면 양고기구이/닭구이/내장구이/어패류구이보다 R02_꼬치구이를 우선한다.
- 불고기 신호가 명확하면 돼지고기구이/소고기구이/오리구이/닭구이보다 R03_불고기를 우선한다.
- 립/바베큐 신호가 명확하면 돼지고기구이/소고기구이/양고기구이/오리구이/닭구이보다 R04_립,바베큐류를 우선한다.
- 껍데기 신호가 명확하면 돼지고기구이보다 R05_껍데기를 우선한다.
- 오돌뼈/오도독뼈처럼 뼈/연골 자체가 본체이면 돼지고기구이보다 R06_뼈,연골구이를 우선한다.
- 특양/양깃머리/곱창/막창/대창/염통처럼 내장·부속 신호가 명확하면 일반 고기 재료축보다 R07_내장,부속구이류를 우선한다.


## 중요 예외 원칙

- "양깃머리", "특양", "양구이", "소양구이"의 "양"은 양고기가 아니라 소의 위/내장 부위 신호로 본다.
- 따라서 양깃머리/특양/양구이는 R10_양고기구이로 보내지 않고 반드시 R07_내장,부속구이류 / 특양구이로 처리한다.
- 족발, 족발구이, 구운족발, 직화족발, 숯불족발, 불족발은 돼지고기일반부위구이에 포함하지 않는다.
- 현재 구이류 block 안에 족발구이 전용 block은 만들지 않는다.
- 숯불, 직화와 같은 구이 신호가 없는 족발, 보쌈, 수육, 편육은 구이류 본체가 아니므로 R98_OUT_OF_SCOPE로 보낸다.
- 닭갈비 단독, 닭발 단독은 구이류로 자동 처리하지 않는다.
- 닭갈비 단독, 닭발 단독은 일반적으로 볶음류 성격이 강한 후보로 보고 R98_OUT_OF_SCOPE로 보낸다.
- 단, 숯불/직화/구운/구이 신호가 명확한 닭갈비/닭발은 R12_닭구이에서 처리한다.

## R98_OUT_OF_SCOPE

family_group = ""

### INTERNAL_PRIORITY

1. 밥/덮밥/정식/세트/코스
2. 볶음류
3. 찜/조림/탕/전골
4. 튀김/전/무침/회/숙회
5. 직화, 숯불 같은 구이 신호가 없는 족발/보쌈/수육/편육 단독
6. 구이 신호가 없는 닭갈비/닭발/닭똥집류 단독명

- 불고기덮밥, 장어덮밥, 스테이크덮밥, bulgogi rice bowl 등 밥/덮밥 본체

- 곱창볶음, 야채곱창볶음, 오돌뼈볶음, 닭똥집볶음, 닭근위볶음, 모래집볶음, 닭발볶음

- 갈비찜, 장어조림, 닭볶음탕, 곱창전골, 불고기전골, bulgogi hot pot

- 후라이드치킨, 양념치킨 등 튀김류 본체
- 치미창가
- 소스명, 토핑명, 원재료명, 비메뉴

### 출력 규칙

- assigned_block = R98_OUT_OF_SCOPE
- final_generic_name = raw_generic_name
- family_group = ""
- normalization_actions = ["exclude_or_correction"]
- review_status = correction_candidate
