## 공통 처리 원칙

- BLOCK_PRIORITY를 위에서부터 적용한다.
- 특정 R block에 들어간 뒤에는 해당 block의 INTERNAL_PRIORITY를 위에서부터 적용한다.
- raw_generic_name은 2차 프롬프트에서 생성된 후보명이고, final_generic_name은 3차에서 확정하는 분석용 이름이다.
- final_generic_name 후보 목록은 “강제 통합 가능한 대표명 목록”이다.
- raw_generic_name이 block 안의 final_generic_name 후보로 명확히 통합 가능하면 반드시 해당 final_generic_name으로 통합한다.
- final_generic_name 후보에 없더라도, raw_generic_name이 특정 R block/family_group에 명확히 속하면 새로운 넓은 대표명으로 억지 통합하지 않는다.
- 이 경우 final_generic_name = raw_generic_name으로 유지하고, family_group은 해당 R block의 family_group으로 출력하며, normalization_actions = ["broad_merge_review"], review_status = needs_review로 둔다.
- R97_면류복합검토와 R99_REVIEW는 검토용 block이므로 final_generic_name = raw_generic_name으로 유지한다.
- AI가 새로운 assigned_block이나 family_group을 임의 생성하면 안 된다.
- AI가 CATEGORY_BLOCK_RULES에 없는 넓은 final_generic_name을 임의 생성하면 안 된다.

- 면류가 아닌 후보는 R98_OUT_OF_SCOPE로 보낸다.
- 면류는 맞지만 판단이 섞여 자동 통합이 위험한 후보는 R97_면류복합검토로 보낸다.
- 면류 여부, 음식명 여부, 적용 가능한 block 자체가 불명확하면 R99_REVIEW로 보낸다.

- 마라, 로제, 크림, 차돌, 해물, 새우, 우삼겹, 명란, 트러플, 바질 등은 대부분 modifier 또는 attribute 신호이다. 단독 final_generic_name으로 만들지 않는다.
- 라면과 라멘은 반드시 분리한다. 라면은 R03_가공면, 라멘은 R04_일식면이다.

- 소바가 포함되면 모밀/메밀보다 소바를 우선한다.
- 야끼소바, 야키소바, 마제소바, 아부라소바는 소바로 통합하되, attribute에 세부 스타일을 남긴다.
- 모밀소바, 냉모밀소바, 메밀소바, 냉메밀소바는 R04_일식면 / 소바로 처리한다.

- 모밀은 R04_일식면에서 처리하지 않고 R05_국산면에서 처리한다.
- 모밀과 메밀국수는 모두 R05_국산면에 속하지만, final_generic_name은 당장 합치지 않고 각각 모밀, 메밀국수로 분리한다.
- 나중에 분석 단계에서 필요하면 모밀과 메밀국수를 후처리로 합칠 수 있다.
- 메밀/모밀이 들어가도 메밀국수 제외 뒤쪽에 막국수, 냉면, 칼국수 같은 국산면 핵심명이 명확하면 해당 국산면 named-dish를 우선한다.

- 국수류 세부 축은 추가로 늘리지 않는다.
- 단, 약한 국수류를 받는 family-level 성격의 final_generic_name으로 일반국수를 둔다.
- 일반국수는 국수류 전체를 빨아들이는 블랙홀 fallback이 아니며 위에 단계를 순차적으로 처리하고 남는 것을 빨아들인다.
- 김치말이국수, 잔치국수, 비빔국수, 열무국수, 쟁반국수, 메밀국수, 모밀, 멸치국수, 동치미국수, 고기국수, 콩국수, 소면, 막국수, 칼국수, 냉면, 쫄면, 밀면처럼 이미 명확한 후보는 일반국수로 통합하지 않는다.

- 도삭면은 중식면 안에서 면종/제조방식 자체가 강한 specialty signal이다.
- raw_generic_name에 도삭면/도삭이 포함되면 짜장, 짬뽕, 마라, 우육, 토마토계란, 대만, 양육 등의 수식어가 함께 있더라도 final_generic_name = 도삭면을 우선한다.
- 이때 짜장, 짬뽕, 마라, 우육, 토마토계란, 대만, 양육 등은 attribute에 남긴다.

- 골뱅이소면, 백골뱅이소면, 꼬막비빔소면, 비빔소면처럼 소면/면이 주된 식사 형태이면 R05_국산면에서 처리한다.
- 단, 골뱅이무침에 소면이 곁들임 사리처럼 붙은 형태라면 면류로 자동 확정하지 않고 무침류의 신호를 더 강하게 보고 R98_OUT_OF_SCOPE로 보낸다.
- 면이 주체인지, 무침/회/구이/탕이 주체인지 불명확하면 R97_면류복합검토로 보낸다.

## R98_OUT_OF_SCOPE

family_group = ""

목적:
면류가 아니거나 다른 category가 더 적절한 후보를 분리한다.

매칭 조건:
- 면사리, 라면사리, 우동사리, 칼국수사리, 당면사리, 사리추가, 면추가, 소스추가, 토핑추가, 비메뉴 설명문
- 리조또, 빠에야, 덮밥, 보울, 볶음밥처럼 밥류 본체
- 죽, 스프, 스튜처럼 죽/스프류 본체
- 마라탕, 나베, 전골, 찌개, 국밥처럼 면보다 국물/밥/전골 본체가 강한 후보
- 완탕, 새우완탕처럼 완탕 단독/국물형 완탕 메뉴
- 짬뽕탕, 누룽지탕, 해물누룽지탕, 계란토마토탕처럼 국,탕류 본체가 강한 후보
- 치킨, 돈까스, 튀김, 구이, 회, 샐러드, 무침처럼 면 본체가 없는 후보
- 커피, 에이드, 맥주, 와인 등 음료/주류
- 크림, 로제, 토마토, 바질, 소스처럼 소스명 또는 재료명 단독 후보

주의 규칙:
- 완탕 단독은 국,탕류로 이동하지만, 완탕면은 면이 본체이므로 R02_중식면 / 완탕면으로 처리한다.
- 짬뽕탕은 국,탕류로 이동하지만, 짬뽕/해물짬뽕/나가사키짬뽕/냉짬뽕은 R02_중식면 / 짬뽕으로 처리한다.
- 샐러드 단독은 면류가 아니지만, 샐러드파스타, COLD PASTA SALAD처럼 파스타/면이 본체이면 R01_양식면 / 파스타로 처리한다.
- 묵사발은 면류가 아니므로 면류에서 제외한다. 단, 묵무침과 다르므로 나물,무침류로 자동 이동하지 않고 기타 또는 국,탕류 검토 후보로 둔다.
- 도삭면사리, 도삭면추가처럼 사리/옵션이면 R98_OUT_OF_SCOPE로 보낸다.
- 면이 단순 사리/곁들임이고 다른 음식이 본체이면 R98_OUT_OF_SCOPE 또는 R97_면류복합검토로 보낸다.

출력 규칙:
- assigned_block = R98_OUT_OF_SCOPE
- final_generic_name = raw_generic_name
- family_group = ""
- normalization_actions = ["exclude_or_correction"]
- review_status = correction_candidate

## R01_양식면

family_group = 양식면

final_generic_name 후보:
- 파스타
- 뇨끼
- 라자냐

INTERNAL_PRIORITY:
1. 라자냐
2. 뇨끼
3. 파스타

bucket 판단:
- 파스타라는 단어가 명확히 있으면 final_generic_name = 파스타로 통합한다.
- 스파게티, 스파게티니, 펜네, 링귀니, 리가토니, 파케리, 페투치네, 탈리아텔레, 파파르델레, 라비올리, 까넬로니 등 양식 파스타 면종이 명확하면 final_generic_name = 파스타로 통합한다.
- 까르보나라, 봉골레, 알리오올리오, 볼로네제, 뽀모도로, 아라비아따, 푸타네스카, 페스카토레, 알프레도, 나폴리탄, 라구처럼 파스타 메뉴명으로 굳은 후보는 final_generic_name = 파스타로 통합한다.
- 뇨끼/뇨키/gnocchi는 파스타로 뭉개지 않고 final_generic_name = 뇨끼로 통합한다.
- 라자냐/라자니아/라쟈냐/lasagna는 파스타로 뭉개지 않고 final_generic_name = 라자냐로 통합한다.
- 샐러드파스타, 냉파스타, 콜드파스타, COLD PASTA SALAD는 샐러드가 아니라 양식면으로 처리한다.
- 양식면은 명확하지만 위 final_generic_name 후보로 통합하기 어려운 후보는 final_generic_name = raw_generic_name으로 유지하고 needs_review로 둔다.

통합 규칙:
- 파스타, 스파게티, 스파게티니, 링귀니, 펜네, 리가토니, 파케리, 카사레치아, 페투치네, 훼투치네, 탈리아텔레, 딸리아뗄레, 파파르델레, 빠빠델레, 라비올리, 까넬로니, 오레키에테, 까르보나라, 봉골레, 알리오올리오, 알리올리오, 볼로네제, 뽀모도로, 포모도로, 아라비아따, 아라비아타, 푸타네스카, 페스카토레, 알프레도, 나폴리탄, 라구
  → final_generic_name = 파스타

- 뇨끼, 뇨키, gnocchi
  → final_generic_name = 뇨끼

- 라자냐, 라자니아, 라쟈냐, lasagna
  → final_generic_name = 라자냐

주의 규칙:
- 크림, 로제, 오일, 토마토, 바질, 명란, 해산물, 새우, 차돌, 트러플 단독은 파스타로 자동 확정하지 않는다.
- 스프카레 파스타, 스튜 파스타처럼 죽/스프류 단어가 있어도 파스타가 본체이면 R01_양식면 / 파스타로 처리한다.
- 리조또가 본체이면 밥류로 이동한다.
- 쌀국수파스타, 미고랭파스타처럼 다른 cuisine의 named-dish와 파스타가 결합되면 R97_면류복합검토로 보낸다.

attribute 규칙:
- 까르보나라, 봉골레, 알리오올리오, 볼로네제, 뽀모도로, 아라비아따, 로제, 크림, 오일, 토마토, 라구, 해산물, 새우, 명란, 트러플, 바질, 리가토니, 링귀니, 펜네 등 세부 소스/면종/재료는 attribute에 남긴다.

bucket 판단:

- 짜장/자장 계열은 도삭면 신호가 없을 때 짜장면으로 통합한다.
- 짬뽕 계열은 도삭면 신호가 없을 때 짬뽕으로 통합한다.
- 완당면은 완탕면으로 통합한다.
- 완탕 단독은 국,탕류 이동 후보이지만, 완탕면은 면이 본체이므로 R02_중식면에서 처리한다.
- 탄탄멘/단단면은 탄탄면으로 통합한다.
- 차오미엔/차오멘은 차우면으로 통합한다.
- 유포멘/유포미엔은 유포면으로 통합한다.
- 울면은 한국식 중화요리 맥락이 강하므로 중식면으로 처리한다.
- 중식면은 명확하지만 위 final_generic_name 후보로 통합하기 어려운 후보는 final_generic_name = raw_generic_name으로 유지하고 needs_review로 둔다.

- 분짜
  → final_generic_name = 분짜

- 미고랭
  → final_generic_name = 미고랭

- 락사
  → final_generic_name = 락사

주의 규칙:
- 똠얌누들, 똠양누들, 꾸어이띠어우, 꿰이띠아오처럼 후보 목록에는 없지만 동남아면임이 명확한 후보는 R06_동남아면에 배정하되 final_generic_name = raw_generic_name으로 유지하고 needs_review로 둔다.
- 미고랭파스타, 쌀국수파스타처럼 동남아면과 양식면이 결합된 후보는 R97_면류복합검토로 보낸다.
- 비리아누들, 멕시칸누들처럼 동남아 외 해외 누들 후보는 올바른 block에서 판단하거나 R97_면류복합검토로 보낸다.

attribute 규칙:
- 소고기, 양지, 해산물, 갈비, 갑오징어, 굴, 곱창, 새우, 치킨, 똠얌, 꾸어이띠어우 등 세부 재료와 스타일은 attribute에 남긴다.

## R97_면류복합검토

family_group = 면류복합검토

목적:
면류 안에는 남을 수 있지만, 기존 R01~R06의 family_group 또는 final_generic_name 후보로 자동 통합하면 의미 손실이나 cuisine 혼동이 생길 수 있는 후보를 검토용으로 분리한다.

INTERNAL_PRIORITY:
1. 볶음면/온면/당면/탕면 계열
2. 칼국수+수제비/만두 등 국산 복합면
3. 면이 주체인지 무침/회/탕/구이 등이 주체인지 애매한 후보
4. 서로 다른 cuisine/named-dish가 결합된 후보
5. family_group 자체가 애매한 누들류

매칭 조건:
- 볶음면, 마라볶음면, 굴소스볶음면, 온면, 마라온면, 옥수수온면, 당면, 중국당면, 넓적당면, 탕면, 토마토탕면
- 칼제비, 칼만두, 육칼, 장칼
- 에그누들, 갈릭누들, 비리아누들, 멕시칸누들처럼 family_group 자체가 애매한 누들류
- 붓카케/붓가케 단독처럼 면류 맥락은 있으나 생략된 핵심명이 불명확한 후보
- 메밀/모밀/소바/국수/막국수/냉면 신호가 섞여 final_generic_name 판단이 애매한 후보

제외 조건:
- 파스타/뇨끼/라자냐 신호가 명확하면 R01
- 중식면 named-dish 신호가 명확하면 R02
- 도삭면/도삭 신호가 명확하면 R02_중식면 / 도삭면
- 라면 신호가 명확하면 R03
- 우동/라멘/소바 신호가 명확하면 R04
- 모밀/메밀국수 신호가 명확하면 R05
- 국산면 named-dish 신호가 명확하면 R05
- 동남아면 named-dish 신호가 명확하면 R06
- 면류가 아님이 명확하면 R98
- 면류 여부 자체가 불명확하면 R99

출력 규칙:
- assigned_block = R97_면류복합검토
- final_generic_name = raw_generic_name
- family_group = 면류복합검토
- normalization_actions = ["broad_merge_review"]
- review_status = needs_review

## R99_REVIEW

family_group = ""

목적:
면류 여부, 음식명 여부, 또는 적용 가능한 R block 자체가 불명확한 후보를 최종 검토한다.

매칭 조건:
- 어떤 block에도 안정적으로 들어가지 않는 경우
- raw_generic_name만으로 면류인지 알 수 없는 경우
- 음식명인지 옵션/소스/재료명인지 불명확한 경우
- 크림, 로제, 토마토, 바질, 생면, 특선처럼 면 요리명으로 확정하기 어려운 단독 후보
- 정체불명의 외국어 메뉴명 또는 오타가 심해 면류 여부 판단이 어려운 후보
- 브랜드명 또는 업장 고유명처럼 보이나 면류 본체가 드러나지 않는 후보

출력 규칙:
- assigned_block = R99_REVIEW
- final_generic_name = raw_generic_name
- family_group = ""
- normalization_actions = ["broad_merge_review"]
- review_status = needs_review
