## 공통 처리 원칙

- BLOCK_PRIORITY를 위에서부터 적용한다.
- 특정 R block에 들어간 뒤에는 해당 block의 INTERNAL_PRIORITY를 위에서부터 적용한다.
- raw_generic_name은 2차 프롬프트에서 생성된 후보명이고, final_generic_name은 3차에서 확정하는 분석용 이름이다.
- final_generic_name 후보 목록은 “강제 통합 가능한 대표명 목록”이다.
- raw_generic_name이 block 안의 final_generic_name 후보로 명확히 통합 가능하면 반드시 해당 final_generic_name으로 통합한다.
- final_generic_name 후보에 없더라도, raw_generic_name이 특정 R block/family_group에 명확히 속하면 새로운 넓은 대표명으로 억지 통합하지 않는다.
- 이 경우 final_generic_name = raw_generic_name으로 유지하고, family_group은 해당 R block의 family_group으로 출력하며, normalization_actions = ["broad_merge_review"], review_status = needs_review로 둔다.
- R97_밥류복합검토와 R99_REVIEW는 검토용 block이므로 final_generic_name = raw_generic_name으로 유지한다.
- AI가 새로운 assigned_block이나 family_group을 임의 생성하면 안 된다.
- AI가 CATEGORY_BLOCK_RULES에 없는 넓은 final_generic_name을 임의 생성하면 안 된다.
- 밥류가 아닌 후보는 R98_OUT_OF_SCOPE로 보낸다.
- 단, 밥류 본체가 명확한 후보는 R98_OUT_OF_SCOPE로 보내지 않는다.
- 일식밥류, 카레밥류, 중식밥류, 동남아볶음밥류, 양식밥류는 일반 볶음밥/일반 덮밥보다 먼저 처리한다.
- 일반덮밥류는 가장 마지막 흡수 block이다.
- 덮밥이라는 단어가 있어도 일식동/회덮밥/카레덮밥/중식덮밥/양식밥류에 명확히 속하면 일반덮밥류로 보내지 않는다.
- 초밥과 스시는 서로 분리한다.
- 초밥 계열은 final_generic_name = 초밥으로, 스시 계열은 final_generic_name = 스시로 출력한다.
- 군함, 군함말이, 군함초밥, 보우즈시, 바즈시, 즈시, 치라시즈시는 초밥 계열로 보고 final_generic_name = 초밥으로 통합한다.
- 마끼류는 별도 block을 만들지 않고 R01_일식밥류 안에서 처리한다.
- 후토마끼, 후토마키, 대왕후토마끼, 대왕후토마키, 후토마끼롤, 마끼, 마키, 김마끼, 야사이마끼, 간뾰마끼, 네기토로마끼, 테마끼, 데마끼 등 마끼 계열은 모두 final_generic_name = 마끼로 통합한다.
- 후토마끼는 별도 final_generic_name으로 보존하지 않고 마끼로 흡수한다.
- 마끼류는 김밥류가 아니라 일식밥류에서 처리한다.
- 사케동, 카이센동, 텐동, 규동, 가츠동처럼 명확한 일식동은 일반덮밥류보다 먼저 R01_일식밥류에서 처리한다.
- 명확한 일식동 계열이지만 세부 final_generic_name 후보로 두지 않은 경우 final_generic_name = 일식동으로 통합한다.
- 단, “동” 한 글자만으로 일식동 처리하지 않는다.
- 카레/커리 계열은 카레밥류에서 final_generic_name = 카레라이스로 통합한다.
- 단, 카레/커리가 들어가도 리조또, 도리아, 가츠동, 텐동, 볶음밥, 솥밥처럼 다른 밥류 본체가 명확하면 카레라이스보다 해당 본체를 우선한다.
- 오므카레/오믈렛카레/오무카레처럼 카레가 본체이고 오므/오믈렛/계란이 스타일 또는 토핑으로 붙은 후보는 R02_카레밥류에서 final_generic_name = 카레라이스로 처리한다.
- 카레오므라이스/커리오므라이스/오므라이스카레처럼 오므라이스 단어가 명확한 후보는 R18_오므라이스류에서 final_generic_name = 오므라이스로 처리한다.
- 이 경우 카레/커리는 final_generic_name으로 만들지 않고 attribute에 남긴다.
- 양식밥류는 리조또/필라프/도리아/잠발라야뿐 아니라 빠에야/포케/부리또볼/타코라이스 등 기타 해외 밥요리도 포함한다.
- 비리야니는 R05_양식밥류가 아니라 R04_동남아볶음밥류에서 처리한다.
- 중식밥류는 짜장밥/짬뽕밥을 보존하고, 나머지 중식식 밥/덮밥/볶음밥 계열은 중식밥으로 통합한다.
- 동남아볶음밥류는 카오팟/나시고랭/비리야니를 보존하고, 나머지 동남아식 볶음밥 계열은 동남아볶음밥으로 통합한다.
- 비빔밥류와 백반류는 분리한다.
- 국밥류와 온반류는 분리한다.
- 김밥류, 주먹밥류, 쌈밥류는 분리한다.
- 누룽지류와 술밥류는 분리한다. 누룽지술밥처럼 술밥 신호가 명확하면 술밥류를 우선한다.
- 알밥류와 솥밥류는 분리한다.
- 밥류 안에는 남을 수 있지만 자동 통합하면 의미 손실이나 category 혼동이 생기는 후보는 R97_밥류복합검토로 보낸다.
- 밥류 여부, 음식명 여부, 적용 가능한 block 자체가 불명확하면 R99_REVIEW로 보낸다.


## R98_OUT_OF_SCOPE

family_group = ""

목적:
밥류가 아니거나 다른 category가 더 적절한 후보를 분리한다.

매칭 조건:
- 면, 국수, 라멘, 우동, 소바, 파스타처럼 면류 본체
- 탕, 국, 찌개, 전골, 샤브샤브처럼 국물 요리 본체
- 구이, 찜, 볶음, 튀김, 전, 부침, 무침, 회처럼 밥류가 아닌 조리법 본체
- 샐러드, 롤, 타코, 버거, 샌드위치, 피자처럼 밥류가 아닌 메뉴 본체
- 디저트, 음료, 소스, 토핑, 추가, 옵션, 비메뉴 설명문
- 세트, 코스, 정식처럼 구성 상품 본체가 강하고 밥 메뉴 본체가 불명확한 후보

bucket 판단:
- 일식밥류는 초밥/스시/마끼/오니기리/일식 덮밥류를 일반밥류보다 먼저 처리하는 block이다.
- 초밥과 스시는 분리한다.
- raw_generic_name에 초밥이 명확하면 final_generic_name = 초밥으로 처리한다.
- raw_generic_name에 스시 또는 sushi가 명확하고 초밥이라는 단어가 없으면 final_generic_name = 스시로 처리한다.
- 군함, 군함말이, 군함초밥, 보우즈시, 바즈시, 즈시, 치라시즈시는 초밥 계열로 보고 final_generic_name = 초밥으로 처리한다.
- 초밥과 스시가 동시에 강하게 들어가 어느 쪽이 본체인지 불명확하면 final_generic_name = raw_generic_name으로 유지하고 needs_review로 둔다.
- 후토마끼/후토마키는 일반 마끼와 같은 축으로 보고 final_generic_name = 마끼로 통합한다.
- 대왕후토마끼, 대왕후토마키, 후토마끼롤처럼 후토마끼 계열 표현이 붙어도 final_generic_name = 마끼로 통합한다.
- 마끼는 김밥류가 아니라 일식밥류 안에서 처리한다.
- 사케동, 카이센동, 텐동, 규동, 가츠동 등 일식동 계열은 일반덮밥류보다 먼저 처리한다.
- 히레카츠동, 로스카츠동 등 카츠동/가츠동 계열은 final_generic_name = 가츠동으로 통합한다.
- 히츠마부시는 장어덮밥이나 일반덮밥으로 보내지 않고 final_generic_name = 히츠마부시로 보존한다.
- 오야코동, 가라아게동, 야키토리동, 야키니쿠동, 스테이크동, 함바그동, 사바동, 멘타이코동, 치라시동, 우나동, 우나기동처럼 일식동 계열은 명확하지만 별도 final_generic_name 후보로 분리하지 않은 경우 final_generic_name = 일식동으로 통합한다.
- 단, “동” 한 글자만으로 일식동 처리하지 않는다.
- 봄동, 안동, 명동처럼 지명/재료/일반 단어에 포함된 “동”은 일식동 신호로 보지 않는다.
- 일식밥류는 명확하지만 아래 final_generic_name 후보 또는 일식동으로 확정하기 어렵다면 final_generic_name = raw_generic_name으로 유지하고 needs_review로 둔다.

- 오므카레/오믈렛카레/오무카레처럼 카레 본체에 오므/오믈렛/계란 스타일이 붙은 후보는 R02_카레밥류로 보낸다.

통합 규칙:
- 오므라이스, 오므라이스덮밥, 오무라이스, 오므밥, 카레오므라이스, 커리오므라이스, 오므라이스카레
  → final_generic_name = 오므라이스

주의 규칙:
- 오므카레/오믈렛카레/오무카레는 R18_오므라이스류가 아니라 R02_카레밥류로 보낸다.
- 카레오므라이스/커리오므라이스/오므라이스카레처럼 오므라이스 단어가 명확한 경우에만 R18_오므라이스류로 처리한다.
- 오믈렛 단독은 밥류가 아니면 다른 category로 보낸다.

attribute 규칙:
- 소스, 치즈, 함박, 돈까스, 카레, 커리 등 세부 정보는 attribute에 남긴다.

## R20_일반덮밥류

family_group = 일반덮밥류

final_generic_name 후보:
- 덮밥

INTERNAL_PRIORITY:
1. 덮밥

bucket 판단:
- 일반덮밥류는 마지막 흡수 block이다.
- 앞선 일식밥류, 카레밥류, 중식밥류, 동남아볶음밥류, 양식밥류, 회덮밥류 등에 해당하지 않는 일반 덮밥 후보를 final_generic_name = 덮밥으로 통합한다.

- 단, 다른 block 신호가 명확하면 일반덮밥류보다 해당 block을 우선한다.

통합 규칙:
- 덮밥, 제육덮밥, 불고기덮밥, 장어덮밥, 낙지덮밥, 오징어덮밥, 스테이크덮밥, 치킨마요덮밥, 소고기덮밥, 돼지고기덮밥, 마요덮밥, 튀김덮밥
  → final_generic_name = 덮밥

주의 규칙:
- 사케동/카이센동/가츠동/텐동/규동/마구로동/우니동/일식동은 R01_일식밥류 우선이다.
- 회덮밥은 R19_회덮밥류 우선이다.
- 카레덮밥/커리덮밥은 R02_카레밥류 우선이다.
- 마파두부덮밥/마라덮밥은 R03_중식밥류 우선이다.
- 부리또볼/라이스볼/타코라이스는 R05_양식밥류 우선이다.
- 비리야니는 R04_동남아볶음밥류 우선이다.
- 덮밥인지 정식/세트인지 불명확하면 R97_밥류복합검토로 보낸다.

attribute 규칙:
- 제육, 불고기, 장어, 낙지, 오징어, 스테이크, 치킨마요, 소고기, 돼지고기, 튀김 등 세부 재료와 스타일은 attribute에 남긴다.


## R97_밥류복합검토

family_group = 밥류복합검토

목적:
밥류 안에는 남을 수 있지만, 기존 R01~R20의 family_group 또는 final_generic_name 후보로 자동 통합하면 의미 손실이나 category 혼동이 생길 수 있는 후보를 검토용으로 분리한다.

INTERNAL_PRIORITY:
1. 여러 밥류 축이 결합된 후보
2. 다른 메뉴 본체와 밥류가 결합된 후보
3. 카레/덮밥/리조또/솥밥 등 block 충돌 후보
4. 밥류 안에 남을 수 있지만 final_generic_name 확정이 어려운 후보
5. final_generic_name 후보로 자동 통합하면 의미 손실이 큰 후보

매칭 조건:
- 해산물솥밥덮밥, 스테이크카레솥밥처럼 솥밥/덮밥/카레가 결합되어 어느 본체를 우선할지 불명확한 후보
- 정식/세트/코스와 밥류가 결합되어 단품 본체가 불명확한 후보
- 밥류는 맞지만 일식밥/양식밥/일반덮밥/볶음밥 중 어디로 보낼지 불명확한 후보
- raw_generic_name만으로 밥요리인지, 반찬/토핑/소스인지 불명확한 후보
- 카레/커리와 다른 밥류 본체가 결합되어 있으나 본체 우선순위를 안정적으로 판단하기 어려운 후보

제외 조건:
- 일식밥류 신호가 명확하면 R01
- 카레밥류 신호가 명확하면 R02
- 중식밥류 신호가 명확하면 R03
- 동남아볶음밥류 신호가 명확하면 R04
- 양식밥류 신호가 명확하면 R05
- 비빔밥/백반/볶음밥/국밥/온반/김밥/주먹밥/쌈밥/누룽지/술밥/알밥/솥밥/오므라이스/회덮밥/일반덮밥 신호가 명확하면 해당 R block으로 보낸다.
- 카레가츠동/카레카츠동은 R01_일식밥류의 가츠동으로 보낸다.
- 커리리조또/카레리조또는 R05_양식밥류의 리조또로 보낸다.
- 커리도리아/카레도리아는 R05_양식밥류의 도리아로 보낸다.
- 비리야니는 R04_동남아볶음밥류로 보낸다.
- 카레볶음밥/커리볶음밥은 R08_볶음밥류로 보낸다.
- 오므카레/오믈렛카레/오무카레는 R02_카레밥류로 보낸다.
- 카레오므라이스/커리오므라이스/오므라이스카레는 R18_오므라이스류로 보낸다.
- 밥류가 아님이 명확하면 R98
- 밥류 여부 자체가 불명확하면 R99

출력 규칙:
- assigned_block = R97_밥류복합검토
- final_generic_name = raw_generic_name
- family_group = 밥류복합검토
- normalization_actions = ["broad_merge_review"]
- review_status = needs_review


## R99_REVIEW

family_group = ""

목적:
밥류 여부, 음식명 여부, 또는 적용 가능한 block 자체가 불명확한 후보를 최종 검토한다.

매칭 조건:
- 어떤 block에도 안정적으로 들어가지 않는 경우
- raw_generic_name만으로 밥류인지 알 수 없는 경우
- 음식명인지 옵션/소스/재료명인지 불명확한 경우
- 오타가 심해 밥류 여부 판단이 어려운 후보
- 브랜드명 또는 업장 고유명처럼 보이나 밥류 본체가 드러나지 않는 후보
- 재료가 불명확하거나 어느 밥류 축에도 확실히 속하지 않는 후보

출력 규칙:
- assigned_block = R99_REVIEW
- final_generic_name = raw_generic_name
- family_group = ""
- normalization_actions = ["broad_merge_review"]
- review_status = needs_review
